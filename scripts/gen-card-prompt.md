# 卡牌生成 Prompt 设计参考

## 来源
SenseNova-U1 官方 prompt 增强文档:https://github.com/OpenSenseNova/SenseNova-U1/blob/main/docs/prompt_enhancement_CN.md

## 核心结论

U1-fast 对**简短 prompt 效果差**是已知的。官方推荐两种方式:

1. **LLM 增强**:用 flash-lite 等先扩写 prompt,再喂给 U1-fast
2. **直接写长篇结构化 prompt**:像设计稿一样精确描述每个区域的位置、内容、颜色、字体、装饰

官方示例(猫咪vs狗狗信息图)的好 prompt 长这样:
- 长度:600+ 字
- 结构:整体布局 → 逐区域描述(位置/内容/颜色/字体/装饰) → 整体风格收束
- 精确到:"画面的正上方居中位置,使用超大号的粗体圆润无衬线字体写着主标题"
- 装饰元素也指定:"中轴线的正中央,放置着一个带有爆炸星芒边缘的亮橙色圆形徽章"

## 三国杀卡牌布局规范(flash-lite 实测官方古锭刀卡面)

### 武器牌(如古锭刀)
- 顶部居中:牌名标题【古锭刀】,黑色书法字,带方头括号
- 左上角:花色点数(扑克牌风格小符号,如 ♠A)——用户要求保留
- 左下角:红色大数字「攻击范围:2」
- 中央:主题插画(古锭刀=单刃短刀,非剑)
- 底部:技能描述原文「锁定技,当你使用【杀】造成伤害时,若指定的目标没有手牌,此伤害+1。」
- 底部标签:「装备·武器」
- 右下角:无(用户明确不要)
- 边框:中式云纹,米黄羊皮纸背景

### 基本牌(如杀)
- 顶部居中:牌名【杀】,红色书法字
- 左上角:花色点数(如 ♠7)
- 中央:武将挥剑插画
- 底部:牌类「基本牌」+ 描述

### 锦囊牌
- 顶部居中:牌名【无中生有】等
- 左上角:花色点数
- 中央:场景插画
- 底部:技能描述 + 「锦囊」标签

## Prompt 模板(长篇段落式布局描述,直接咽 U1-fast)

脚本 `scripts/gen-card.mjs` 的 `buildPrompt()` 现采用此模板。以古锭刀为例:

```
这是一张三国杀卡牌设计图,竖版卡面,整体宽高比约为1:1.4,采用工笔重彩中国古典画风格,画面高质感、细节精细。

画面的正上方居中位置,使用粗体黑色书法字体写着主标题"【古锭刀】",标题带有方头括号【】装饰,字号醒目,占据卡面顶部约10%的区域。

画面的左上角区域,使用扑克牌风格的小符号标注花色点数"♠A",花色符号(黑色)和点数字母(黑色)清晰可辨,仅出现一次,不重复,占据左上角约8%的面积。

画面的左下角区域,使用红色大号数字"2"标注攻击范围,数字右侧用红色小字竖排写着"攻击范围",形成武器牌特有的范围标识,占据左下角约8%的面积。

画面的正中央,占据卡面主体面积约60%的插画区域,绘制一柄古朴的短刀(重点:这是刀不是剑!刀身单刃、略弯曲、有刀柄和十字护手,属于单刃短兵器;不是双刃直剑)。背景衬以淡墨山水或云纹,工笔重彩风格,插画需符合中国古典三国题材。

画面的底部居中区域,使用黑色小号楷体字写着技能描述原文:"锁定技,当你使用【杀】造成伤害时,若指定的目标没有手牌,此伤害+1。"。文字需完整、清晰、无重复、无错字,占据底部约10%的面积。

技能描述文字下方,使用更小的灰色字标注牌类标签"装备·武器"。

画面的四周边框,采用中式云纹和回纹交织的装饰边框,底色为米黄色羊皮纸质感,边框宽度适中,不喧宾夺主。

画面右下角完全留空,不绘制任何花色点数符号或装饰图案。画面右上角也留空。左上角的花色点数符号只出现一次,不要重复堆砌。

整体严格参照三国杀官方卡牌的布局规范,信息层次分明,文字清晰可读,不要在角落堆砌花色符号。
```

## 实测结果(古锭刀)

flash-lite 视觉验证评价(对比官方参考卡):

| 检查项 | 结果 |
|---|---|
| 标题含【】居中 | ✅ 符合 |
| 左上角花色点数 | ✅ A+黑桃符号(已恢复) |
| 左下角攻击范围 | ✅ 红色「2」+ 竖排「攻击范围」|
| 底部技能描述 | ✅ 文字完整(仅"锁定技"误写为"锁定极",模型文字渲染小错) |
| 中央插画是刀非剑 | ✅ "一把刀,单刃,有护手,符合古锭刀特征"(已修正) |
| 右下角冗余花色 | ✅ 无(已修正) |
| 整体相似度 | ✅ 中偏高(布局一致,仅美术风格有差异) |

## 关键优化点总结

1. **长篇描述**:从分号连接短句改为段落式空间布局说明,每段描述一个区域
2. **位置精确**:"正上方居中"、"左上角区域"、"左下角区域"、"正中央"、"底部居中区域"
3. **比例指定**:每个区域标注占卡面面积百分比(10%/8%/60%/10%),引导模型分区渲染
4. **字体指定**:"粗体黑色书法字"、"扑克牌风格小符号"、"红色大号数字"、"黑色小号楷体字"
5. **负面约束明确独立成段**:"右下角完全留空"、"左上角花色点数只出现一次不要重复堆砌"
6. **物体特征强调**:古锭刀插画描述用括号+感叹号强化"这是刀不是剑!"+ 详细刀的特征(单刃/略弯曲/十字护手)
7. **文字原文**:技能描述直接嵌入引号内,要求"完整、清晰、无重复、无错字"
8. **花色颜色按 suit 动态**:suitColorName(s) 根据 ♥♦→红/♠♣→黑 填入
