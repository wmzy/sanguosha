# 交互式选将流程设计方案

## 1. 现状分析

### 当前 bootstrap 流程

```
create(config) → bootstrap(state, config)
  └─ dispatch(开局 start)
       └─ execute:
            1. applyAtom(抽身份)     ← 瞬时，随机分配身份
            2. applyAtom(选将)       ← 瞬时，随机分配武将
            3. instantiateSkill()    ← 注册技能实例
            4. applyAtom(初始化洗牌)  ← 瞬时
            5. applyAtom(发牌)       ← 瞬时
            6. applyAtom(回合开始)   ← 启动第一回合
```

**核心问题**:整个 execute 是一个 async 函数，一气呵成跑完。`选将` atom 是非等待型——apply 直接修改 state，没有 pending 阶段。玩家无法参与。

### 现有 pending/respond 机制

引擎已有成熟的等待-回应模式:

```typescript
// 等待型 atom 示例:询问闪
{
  type: '询问闪',
  apply(_state) { /* 不做事——等待型 */ },
  pending: {
    onTimeout: { type: '无操作' },
    prompt: { type: 'useCard', title: '是否出闪', ... },
    timeout: 15,
  },
}
```

当 `applyAtom` 遇到带 `pending` 的 atom 时:
1. 执行 `apply` (等待型 atom 为空操作)
2. 创建 `PendingSlot` 挂到 `state.pendingSlot`
3. `await new Promise(resolve => ...)` —— execute 挂起
4. 玩家通过 `dispatch({ actionType: 'respond', ... })` 回应
5. respond entry 的 `execute` 被调用 → `resolve()` → 父 execute 恢复

**关键洞察**:这个机制完全可以复用于选将。只需要:
- 选将是等待型 atom（用 `请求回应` 复用）
- 开局 execute 内部循环等待每个玩家
- 注册临时 respond entry 处理选择结果

## 2. 整体架构设计

### 2.1 拆分后的流程

```
bootstrap(state, config)
  └─ dispatch(开局 start)
       └─ execute:
            ┌─── 阶段 1:初始化(瞬时) ───┐
            │ 1. applyAtom(抽身份)        │
            │ 2. 初始化选将候选池          │
            └────────────────────────────┘
                    ↓
            ┌─── 阶段 2:选将(等待型) ───┐
            │ for each player:            │
            │   applyAtom(请求回应)       │ ← 挂起，等玩家选
            │   玩家 respond → 读结果     │
            │   设置武将+技能              │
            └────────────────────────────┘
                    ↓
            ┌─── 阶段 3:启动游戏(瞬时) ──┐
            │ 3. instantiateSkill()       │
            │ 4. applyAtom(初始化洗牌)     │
            │ 5. applyAtom(发牌)          │
            │ 6. applyAtom(回合开始)      │
            └────────────────────────────┘
```

### 2.2 状态管理

选将期间的临时数据存储在 `state.localVars` 中:

```typescript
// 选将候选池(所有玩家共享，选完一个减一个)
state.localVars['__选将候选池'] = [
  { name: '刘备', skills: ['仁德'] },
  { name: '曹操', skills: ['护甲'] },
  { name: '孙权', skills: ['制衡'] },
  // ...
];

// 选将进度(供其他玩家查看)
state.localVars['__选将进度'] = {
  order: [0, 3, 1, 2, 4],  // 选择顺序(主公优先)
  current: 0,                // 当前第几个选
  selected: {                // 已选结果
    0: '刘备',
    3: '曹操',
  },
};
```

**为什么用 `localVars` 而不是新字段?**
- `localVars` 已有——`GameState` 定义了 `localVars: Record<string, Json>`
- 临时数据，选将结束后清理
- 不需要修改 `GameState` 类型
- 通过 `buildView` 自动暴露给前端

## 4. 核心代码修改

### 4.1 修改 选将 atom → 选将准备（非等待型）

将现有 `选将` atom 重命名为 `选将准备`，只负责初始化候选池，不负责分配:

```typescript
// src/engine/atoms/选将.ts — 替换现有 选将 atom

export const 选将准备: AtomDefinition<{
  type: '选将准备';
  characters: Array<{ name: string; skills: string[] }>;
  seed: number;
  /** 每个玩家可选武将数（默认 5） */
  candidatesPerPlayer?: number;
}> = {
  type: '选将准备',
  validate(state) {
    if (state.players.length === 0) return '没有玩家';
    return null;
  },
  apply(state, atom) {
    const { characters, seed, candidatesPerPlayer = 5 } = atom;
    const rng = createRng(seed);
    const playerCount = state.players.length;

    // 打乱武将池
    const pool = [...characters];
    for (let i = pool.length - 1; i > 0; i--) {
      const j = rng.nextInt(i + 1);
      [pool[i], pool[j]] = [pool[j], pool[i]];
    }

    // 确定选择顺序:主公先选，其余随机
    const lordIdx = state.players.findIndex(p => p.vars['身份'] === '主公');
    const others = state.players
      .filter((_, i) => i !== lordIdx)
      .map(p => p.index);
    // 打乱非主公玩家顺序
    for (let i = others.length - 1; i > 0; i--) {
      const j = rng.nextInt(i + 1);
      [others[i], others[j]] = [others[j], others[i]];
    }
    const order = lordIdx >= 0 ? [lordIdx, ...others] : others;

    // 存入 localVars
    state.localVars['__选将候选池'] = pool as unknown as Json;
    state.localVars['__选将进度'] = {
      order,
      current: 0,
      selected: {},
    } as unknown as Json;
    state.localVars['__选将每人候选数'] = candidatesPerPlayer as unknown as Json;
  },
};
```

### 4.2 修改 开局 skill 的 execute

```typescript
// src/engine/skills/开局.ts — 核心修改

import { applyAtom, pushNotify } from '../create-engine';
import {
  registerActionEntry,
  unregisterActionEntry,
  instantiateSkill,
} from '../skill';

const SYSTEM_OWNER = -1;

interface GameConfig {
  characters: Array<{ name: string; skills: string[] }>;
  playerCount: number;
  seed: number;
  handSize?: number;
  /** 每人可选武将数（默认 5） */
  candidatesPerPlayer?: number;
}

export function createSkill(id: string, ownerId: number): Skill {
  return {
    id,
    ownerId,
    name: '开局',
    description: '游戏开局:抽身份、选将、洗牌、发牌、启动第一回合',
  };
}

export function onInit(_skill: Skill, _config: unknown): () => void {
  const entry: ActionEntry = {
    skillId: '开局',
    ownerId: SYSTEM_OWNER,
    actionType: 'start',
    validate: () => null,
    execute: async (state: GameState, params: Record<string, Json>) => {
      const config = params as unknown as GameConfig;
      const { characters, playerCount, seed, handSize = 4, candidatesPerPlayer = 5 } = config;

      // ──── 阶段 1:初始化(瞬时) ────

      // 1. 抽身份(每人一张,主公亮明)
      await applyAtom(state, { type: '抽身份', playerCount, seed });

      // 2. 准备选将候选池(不分配武将)
      await applyAtom(state, {
        type: '选将准备',
        characters,
        seed,
        candidatesPerPlayer,
      });

      // ──── 阶段 2:交互式选将 ────

      // 注册临时 respond entry（选将结束后注销）
      registerSelectCharacterEntry(state);

      const progress = state.localVars['__选将进度'] as {
        order: number[];
        current: number;
        selected: Record<number, string>;
      };
      const pool = state.localVars['__选将候选池'] as Array<{ name: string; skills: string[] }>;
      const pickCount = candidatesPerPlayer;

      for (let i = 0; i < progress.order.length; i++) {
        const playerIdx = progress.order[i];
        const player = state.players[playerIdx];
        if (!player || !player.alive) continue;

        // 更新进度
        progress.current = i;

        // 为当前玩家生成候选列表（从池中抽取 pickCount 个）
        // 已被选走的武将跳过
        const selectedNames = new Set(Object.values(progress.selected));
        const available = pool.filter(c => !selectedNames.has(c.name));
        const candidates = available.slice(0, pickCount);

        if (candidates.length === 0) continue;

        // 存储当前玩家的候选列表（供 respond 校验用）
        state.localVars[`__选将候选_${playerIdx}`] = candidates as unknown as Json;

        // 推送 notify 事件（其他玩家看到"XX 正在选将"）
        pushNotify(state, {
          skillId: '开局',
          eventType: '选将开始',
          data: { player: playerIdx, candidates: candidates.map(c => c.name) },
        });

        // 等待玩家选择（请求回应 atom，挂起直到 respond/超时）
        await applyAtom(state, {
          type: '请求回应',
          requestType: '__选将',
          target: playerIdx,
          prompt: {
            type: 'chooseCharacter',
            title: `请选择武将（${i + 1}/${progress.order.length}）`,
            description: `你是${player.vars['身份'] as string ?? '未知身份'}`,
            candidates,
            pick: 1,
          },
          timeout: 60,
        });

        // respond 已执行——读取结果
        // （respond entry 将选择结果写入 progress.selected）
        const selectedName = progress.selected[playerIdx];
        if (selectedName) {
          const char = pool.find(c => c.name === selectedName);
          if (char) {
            player.character = char.name;
            player.skills = [...char.skills];
          }
        }

        // 清理临时候选数据
        delete state.localVars[`__选将候选_${playerIdx}`];
      }

      // 注销临时 respond entry
      unregisterSelectCharacterEntry();

      // 清理选将临时数据
      delete state.localVars['__选将候选池'];
      delete state.localVars['__选将进度'];
      delete state.localVars['__选将每人候选数'];

      // ──── 阶段 3:启动游戏(瞬时) ────

      // 2.5 注册技能实例
      for (const player of state.players) {
        for (const skillId of player.skills) {
          await instantiateSkill(skillId, player.index);
        }
      }

      // 3. 初始化洗牌
      await applyAtom(state, { type: '初始化洗牌', seed });

      // 4. 发牌(主公多摸 1 张)
      await applyAtom(state, { type: '发牌', handSize, lordBonus: 1 });

      // 5. 启动第一回合(从主公开始)
      const lord = state.players.find(p => p.vars.身份 === '主公');
      if (lord) {
        await applyAtom(state, { type: '回合开始', player: lord.index });
        await applyAtom(state, { type: '阶段开始', player: lord.index, phase: '准备' });
        await applyAtom(state, { type: '阶段结束', player: lord.index, phase: '准备' });
      }
    },
  };

  registerActionEntry(entry);
  return () => unregisterActionEntry('开局', SYSTEM_OWNER, 'start');
}

// ──── 临时 respond entry ────

function registerSelectCharacterEntry(state: GameState): void {
  const entry: ActionEntry = {
    skillId: '开局',
    ownerId: SYSTEM_OWNER,
    actionType: 'respond',
    validate(_state, params) {
      const { character } = params;
      if (typeof character !== 'string' || !character) return '请选择武将';
      return null;
    },
    execute(state, params) {
      const { character } = params;
      const progress = state.localVars['__选将进度'] as {
        order: number[];
        current: number;
        selected: Record<number, string>;
      };
      if (!progress) return;

      const playerIdx = progress.order[progress.current];
      if (playerIdx === undefined) return;

      // 校验:武将必须在候选列表中
      const candidates = state.localVars[`__选将候选_${playerIdx}`] as Array<{ name: string }>;
      if (!candidates || !candidates.some(c => c.name === character)) return;

      // 记录选择结果
      progress.selected[playerIdx] = character;
    },
  };
  registerActionEntry(entry);
}

function unregisterSelectCharacterEntry(): void {
  unregisterActionEntry('开局', SYSTEM_OWNER, 'respond');
}
```

### 4.3 选将超时处理

当玩家超时未选时，`请求回应` 的 `pending.onTimeout` 触发。需要一个自动选将的 atom:

```typescript
// src/engine/atoms/选将超时.ts（新文件）

import type { AtomDefinition, Json } from '../types';
import { registerAtom } from '../atom';
import { createRng } from '../../shared/rng';

/** 选将超时:从候选列表中随机选一个 */
export const 选将超时: AtomDefinition<{ type: '选将超时'; player: number }> = {
  type: '选将超时',
  validate(state, atom) {
    if (!state.players[atom.player]) return '玩家不存在';
    return null;
  },
  apply(state, atom) {
    const progress = state.localVars['__选将进度'] as {
      order: number[];
      current: number;
      selected: Record<number, string>;
    };
    if (!progress) return;

    const playerIdx = progress.order[progress.current];
    if (playerIdx === undefined || playerIdx !== atom.player) return;

    const candidates = state.localVars[`__选将候选_${playerIdx}`] as Array<{ name: string }>;
    if (!candidates || candidates.length === 0) return;

    // 随机选一个
    const rng = createRng(Date.now());
    const pick = candidates[rng.nextInt(candidates.length)];
    progress.selected[playerIdx] = pick.name;
  },
};

registerAtom(选将超时);
```

然后修改 `请求回应` 的 `__选将` 场景的超时配置。由于 `请求回应` 是通用 atom，其 `pending.onTimeout` 固定为 `{ type: '无操作' }`。我们需要在 `buildView` 或 session 层面处理。

**更简单的方案**:让 `请求回应` 的 `pending` 配置支持动态 `onTimeout`。但当前 `AtomPending` 的 `onTimeout` 是静态的。

**实际方案**:在 `选将超时` atom 注册后，修改 `请求回应` atom 的 `apply` 来检测 `requestType === '__选将'`:

```typescript
// 替代方案:在请求回应 atom 中特殊处理 __选将 超时
// 但这违反了"通用 atom 不做特殊逻辑"的原则

// 最佳方案:在 applyAtom 管线中，当 pending.onTimeout 是 '无操作' 且
// requestType='__选将' 时，替换为 { type: '选将超时', player: target }
```

**推荐的最简方案**:在 `applyAtom` 创建 pending 时，根据 `requestType` 动态决定 `onTimeout`:

```typescript
// src/engine/create-engine.ts — applyAtom 中 pending 创建部分

if (def.pending) {
  await new Promise<void>((resolve) => {
    const pending = def.pending!;
    // ... existing code ...

    // 动态 onTimeout:选将超时自动随机选
    let effectiveOnTimeout = pending.onTimeout;
    const reqType = (current as { requestType?: string }).requestType;
    if (reqType === '__选将' && pending.onTimeout.type === '无操作') {
      const target = (current as { target?: number }).target;
      if (typeof target === 'number') {
        effectiveOnTimeout = { type: '选将超时', player: target };
      }
    }

    // ... 用 effectiveOnTimeout 替代 pending.onTimeout ...
  });
}
```

**但这个方案侵入性太强。** 更好的做法是:

**最终方案**:在 `开局` skill 的 execute 循环中，给 `请求回应` 传自定义的 `onTimeout`。但当前 Atom 联合类型中 `请求回应` 没有 `onTimeout` 字段。

**最干净的方案**:在 `开局` execute 中，手动创建 pending 而不是通过 `applyAtom`。但这破坏了管线。

**最终决定**:在 `请求回应` atom 的 `pending` 中，将 `onTimeout` 改为读取 atom 上的 `timeoutAction` 字段:

```typescript
// src/engine/atoms/请求回应.ts — 修改 pending 配置

pending: {
  // 优先读 atom 上的 onTimeout 字段，fallback 到默认无操作
  onTimeout: { type: '无操作' },  // 默认值，会被 applyAtom 管线覆盖
  prompt: { type: 'confirm', title: '请回应' },
  timeout: 30,
},
```

并在 `applyAtom` 管线中，当 atom 上有 `onTimeout` 字段时覆盖 `pending.onTimeout`:

```typescript
// applyAtom 中 pending 创建处
const atomOnTimeout = (current as { onTimeout?: Atom }).onTimeout;
const effectiveOnTimeout = atomOnTimeout ?? pending.onTimeout;
```

这样 `开局` execute 就可以:

```typescript
await applyAtom(state, {
  type: '请求回应',
  requestType: '__选将',
  target: playerIdx,
  prompt: { type: 'chooseCharacter', ... },
  timeout: 60,
  onTimeout: { type: '选将超时', player: playerIdx },  // ← 自定义超时行为
});
```

## 7. 前端协议层修改

### 7.1 handleAction 适配

前端发送选将响应时，需要走标准的 action 流程:

```typescript
// src/client/components/GameView.tsx

// 选将响应
function handleSelectCharacter(characterName: string) {
  // 以当前玩家(viewer)的 ownerId 发送
  // skillId='开局', actionType='respond', params={ character: '刘备' }
  send('开局', 'respond', { character: characterName });
}
```

### 7.2 Session 层无需修改

Session 的 `handleAction` 已经是通用的:

```typescript
async handleAction(playerId: string, action: EngineClientMessage): Promise<void> {
  // ... 校验 ownerId ...
  void dispatch(this.state, action);
}
```

只要 `action.skillId = '开局'`, `action.actionType = 'respond'`, `action.params = { character: '刘备' }`，dispatch 就能找到我们注册的 respond entry。

## 8. 边界情况处理

### 8.1 超时自动选将

通过 `请求回应` atom 的 `onTimeout` 字段（或 applyAtom 管线的动态覆盖），超时时触发 `选将超时` atom，从候选列表随机选一个。

### 8.2 玩家断线

选将期间断线的玩家:
- 如果超时→自动随机选
- 如果重连→看到选将 UI，可以继续选（pending slot 还在）

### 8.3 restore（从日志恢复）

选将期间的 restore 是已知限制:
- `restore` 不调 `bootstrap`，不注册 `__选将` respond entry
- 解决方案:在 `restore` 中检测 `localVars['__选将进度']` 存在时，调用 `registerSelectCharacterEntry`
- 或者:将 `选将准备` 和每次 `请求回应` 都记入 `actionLog`，restore 时重放

### 8.4 武将去重

同一局游戏中每个武将只能被一个玩家选择。
- `选将准备` atom 打乱整个池
- 循环中用 `progress.selected` 追踪已选武将
- `available = pool.filter(c => !selectedNames.has(c.name))` 保证不重复

### 8.5 主公特殊规则

三国杀规则:主公选将时不知道其他人身份。但主公先选，候选池最大。
- 主公获得 5 个候选（默认）
- 其他玩家也获得 5 个候选
- 未来可扩展:主公显示全部武将供参考
