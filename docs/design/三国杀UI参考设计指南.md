## 1. 整体布局架构

### 1.1 桌面版 (PC/OL) 布局

```
┌─────────────────────────────────────────────────────────────────────┐
│ ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐       │

│ │ 头像+HP │  │ 头像+HP │  │ 头像+HP │  │ 头像+HP │  │ 头像+HP │       │

│  │  P3    │    ┌──────────────────┐   │  P1    │                   │
```

**关键特征:**
- 窗口比例: 宽屏横向 (16:9 或更宽)
- 对手玩家以弧形/半圆形环绕排列在上方和两侧
- 顶部区域: 3-5个对手并排展示
- 两侧区域: 各1个对手
- 中央: 弃牌堆 + 牌堆 + 动作动画
- 底部: 自己的完整面板 + 手牌 + 操作按钮

**移动版适配要点:**
- 装备栏移到右侧(或与武将面板横向排列)
- 两名武将头像合并以扩大手牌显示区
- 手牌支持横向滚动 + 扇形堆叠
- 底部按钮区域固定不随内容滚动
- 对手信息压缩为小头像 + 体力条

### 1.3 4人/5人/8人布局差异

| 人数 | 座位排列 | 说明 |
|------|----------|------|
| 4人 | 矩形四角 | 自己在下方，对手在上左上右 |
| 5人 | 十字形 | 自己在下，4个对手在上、左上、右上、左右 |
| 8人 | 椭圆弧形 | 自己在正下方，7个对手沿弧线排列 |

## 2. 座位布局详细设计

### 2.1 座位排列规则

- **逆时针排序**: 玩家按逆时针方向排列座位号 1→N
- **主公定位**: 主公通常固定在正上方或正对面位置
- **自己定位**: 自己始终在屏幕底部中央
- **视角旋转**: 以自己为中心,其他玩家围绕分布

### 2.2 座位卡信息密度

每个对手座位卡显示:

```
┌──────────────────────────┐
│ 玩家名    座次号    [身份] │  ← 身份仅对已公开者可见
│ ┌────────────────────┐   │
│ │     武将头像        │   │  ← 立绘或头像(圆形/方形)
│ └────────────────────┘   │
│ 武将名: 刘备              │
│ ❤️❤️❤️❤️ (4/4)          │  ← 体力: 红心=满, 灰心=空
│ ⚔️[诸葛连弩] 🛡[八卦阵]   │  ← 装备图标(横排)
│ 🐴[+1马]                 │
│ 手牌: 5张                 │  ← 手牌数(看不到具体内容)
│ 技能: [仁德] [激将]       │  ← 技能标签
└──────────────────────────┘
```

### 2.3 座位卡状态样式

| 状态 | 样式 |
|------|------|
| **当前回合** | 金色边框 + 发光动效 `border: 2px solid #ffd700` |
| **自己视角** | 蓝色边框 `border: 2px solid #3498db` |
| **已死亡** | 透明度 0.35, 灰色遮罩 `opacity: 0.35` |
| **可被选为目标** | 虚线边框 + hover 红色 `cursor: pointer` |
| **已被选为目标** | 红色外描边 `outline: 3px solid #e74c3c` |

```
│  ├─────────────────────┤│
```

### 3.2 武将卡设计规范

| 元素 | 规格 |
|------|------|
| **尺寸** | 标准: 800×1200px (2:3比例) |
| **安全边距** | 10% 内缩, 防裁切 |
| **主图区** | 占卡面 70%, 半身或全身立绘 |
| **技能描述区** | 占 25%, 背景色较深 |
| **字体** | 技能名: 粗体楷书; 正文: 宋体变体; 数值: 无衬线体 |
| **字号层级** | 标题 28pt / 数值 24pt / 正文 16pt |
| **边框装饰** | 势力对应色的古风花纹边框 |

## 4. 手牌区设计

### 4.1 手牌区位置

| 平台 | 位置 | 排列方式 |
|------|------|----------|
| **桌面版** | 屏幕底部, 占全宽 | 扇形展开 或 水平排列 |
| **移动版** | 屏幕底部固定区域 | 横向滚动 + 堆叠 |

### 4.2 手牌卡样式

```
┌──────────────┐
│  ♠            │  ← 花色 (左上角)
│              │
│     杀        │  ← 牌名 (居中, 大字)
│              │
│  伤害:1       │  ← 数值信息
│  [武器图标]    │  ← 牌类型图标
│            A  │  ← 点数 (右下角)
└──────────────┘
```

### 4.3 手牌交互状态

| 状态 | 样式 |
|------|------|
| **默认** | 暗色背景, 可点击 |
| **可使用** | 高亮边框 + 微光动效 |
| **已选中** | 上浮 + 选中高亮 `border: 2px solid #ffd700` |
| **不可用** | 半透明 `opacity: 0.4` |
| **新摸到** | 滑入动画 `drawCardIn 0.45s cubic-bezier(0.23, 1, 0.32, 1)` |
| **弃牌模式** | 红色选中态 |

### 4.4 花色颜色编码

```typescript
const SUIT_COLOR = {
  '♠': '#ccc',   // 黑色 - 灰白 (暗色背景)
  '♣': '#ccc',   // 黑色 - 灰白
  '♥': '#e74c3c', // 红色
  '♦': '#e74c3c', // 红色
};
```

## 5. 装备区设计

### 5.1 装备区位置

| 平台 | 位置 |
|------|------|
| **桌面版** | 武将面板内, 位于体力值下方 |
| **移动版** | 武将面板右侧 或 下方图标行 |

### 5.2 装备槽位 (5个固定位置)

```
┌─────────────────────────────────────────────────┐
│ ⚔️[武器]  🛡[防具]  🐴[+1坐骑]  🐴[-1坐骑]  💎[宝物] │
│  诸葛连弩   八卦阵     绝影        大宛         -      │
└─────────────────────────────────────────────────┘
```

| 槽位 | 图标 | 说明 |
|------|------|------|
| 武器 | ⚔️ 或刀剑图标 | 橙色/金色系 |
| 防具 | 🛡 或盾牌图标 | 蓝色/银色系 |
| +1坐骑 | 🐴 或马匹图标 | 绿色系 (防御向) |
| -1坐骑 | 🐴 或马匹图标 | 红色系 (进攻向) |
| 宝物 | 💎 或宝石图标 | 紫色系 |

### 5.3 装备卡视觉风格

- 卡背: 深色古风纹理
- 卡面: 装备立绘 + 名称 + 效果描述
- 装备区牌 **正面朝上**, 所有人可见

## 6. 判定区设计

### 6.1 判定区位置

判定区通常位于 **武将牌上方或旁边**, 表示延迟效果:

```
┌──────────────────────────┐
│  判定区: [乐不思蜀] [闪电] │  ← 延时锦囊
│  ┌────────────────────┐  │
│  │     武将牌          │  │
│  └────────────────────┘  │
│  装备区: [武器][防具]...  │
└──────────────────────────┘
```

### 6.2 判定区规则

- 判定区牌 **不属于任何玩家**, 但放置在目标玩家处
- 同名延时锦囊不可叠加 (判定区不能有两张同名牌)
- 判定区牌正面朝上, 所有人可见
- 判定结果产生后, 牌进入弃牌堆

### 6.3 延时锦囊视觉标记

| 锦囊 | 视觉特征 |
|------|----------|
| 乐不思蜀 | 方块花色, 沙漏标记, 横向放置 |
| 兵粮寸断 | 黑色基本牌/装备牌外观, 沙漏标记 |
| 闪电 | 特殊闪电图标, 红/黑色系 |

## 7. 身份牌设计

### 7.1 四大身份及颜色

| 身份 | 颜色 | 图案 | 数量 | 胜利条件 |
|------|------|------|------|----------|
| **主公** | 🔴 红色 (#e74c3c) | 头盔 | 1 | 消灭反贼+内奸 |
| **忠臣** | 🟡 黄色 (#f39c12) | 花朵 | 3 | 保护主公, 消灭反贼+内奸 |
| **反贼** | 🟢 绿色 (#2ecc71) | 火焰 | 4 | 杀死主公 |
| **内奸** | 🔵 蓝色 (#3498db) | 眼睛 | 2 | 最后存活 |

### 7.2 身份牌在游戏中的显示

```
┌───────────────┐
│   身份牌       │
│               │
│   ┌─────┐    │
│   │图标 │    │  ← 身份对应的图案
│   └─────┘    │
│               │
│   主公/忠臣/   │  ← 身份文字 (大字)
│   反贼/内奸   │
│               │
│ [底色=身份色] │  ← 整张卡底色为身份色
└───────────────┘
```

### 7.3 身份牌在界面中的使用

- **主公**: 公开身份, 使用主公标记 (👑 图标)
- **其他身份**: 初始隐藏, 死亡后翻开
- **身份标签**: 在座位卡上以小标签形式显示
  - 主公: `lordBadge` 金色
  - 忠臣: `loyalistBadge` 黄色
  - 反贼: `rebelBadge` 绿色
  - 内奸: `renegadeBadge` 蓝色
  - 未公开: `hiddenBadge` 灰色

## 8. 操作按钮区

```
┌──────────────────────────────────────────────┐
│  [出牌]        [发动技能]        [结束回合]   │
```

## 9. 选将界面

### 9.1 选将流程

1. 系统从武将池随机抽取候选武将
2. 主公先选 → 其他玩家依次选择
3. 每人展示 3-5 张候选武将卡
4. 倒计时内点击选择

### 9.2 选将界面布局

```
│  │     │  │     │  │     │  │     │        │

│  │❤️4  │  │❤️4  │  │❤️4  │  │❤️4  │        │
```

### 9.3 选将卡视觉设计

- 候选卡比正式手牌大 (宽度约 150-200px)
- 未选中: 默认暗色
- Hover: 边框高亮 + 微放大
- 已选中: 金色边框 + 上浮效果
- 已被其他玩家选走: 灰色 + 删除线

## 10. 颜色方案

### 10.1 全局配色

```typescript
const GLOBAL_COLORS = {
  // 背景色
  bg: {
    page: '#1a1a2e',      // 页面背景 (深蓝黑)
    panel: '#2c3e50',     // 面板背景 (深灰蓝)
    input: '#34495e',     // 输入框背景
    nav: '#16213e',       // 导航栏 (最深)
    table: '#0d1117',     // 牌桌背景 (纯黑偏蓝)
  },
  
  // 文字色
  text: {
    primary: '#eee',      // 主文字 (亮白)
    secondary: '#bdc3c7', // 次要文字
    muted: '#95a5a6',     // 淡化文字
    dim: '#7f8c8d',       // 更淡文字
  },
  
  // 强调色
  accent: {
    gold: '#ffd700',      // 金色 (当前回合/选中)
    blue: '#3498db',      // 蓝色 (自己视角/按钮)
    red: '#e74c3c',       // 红色 (伤害/取消/选目标)
    green: '#2ecc71',     // 绿色 (确认/满血)
    orange: '#e67e22',    // 橙色 (提示/倒计时)
    purple: '#9b59b6',    // 紫色 (特殊状态)
  },
};
```

## 11. 卡牌美术风格

### 11.1 整体风格

- **国风水墨 + 精美插画**: 融合传统中国水墨画元素与现代游戏立绘
- **暗色主题**: 牌桌和背景使用深色调, 突出卡牌和武将
- **古典建筑美学**: 背景融入古代建筑、山水、战场等元素

### 11.2 卡牌视觉层次

```
第1层: 底色/渐变背景     ← 势力色深色
第2层: 古风边框/纹样     ← 金色/铜色装饰线
第3层: 角色立绘/插画     ← 武将全身/半身像
第4层: 文字信息层        ← 名字、技能、数值
第5层: 状态标记层        ← buff/debuff 图标
```

### 11.3 牌背设计

- 统一牌背: 深红/暗金色, 太极或祥云纹样
- 中心: 三国杀 Logo 或太极图案
- 边框: 金色装饰线

### 11.4 牌面信息层次

```
┌────────────────────────┐
│ 花色 点数     ← 左上角   │  小字, 不抢视觉焦点
│                        │
│                        │
│    [立绘/插画]         │  ← 主视觉, 占最大面积
│                        │
│                        │
│ ─────────────────────  │  ← 分割线
```

## 14. 设计规律总结

### 14.1 信息层次规律

1. **最重要的信息最大最亮**: 武将名、体力值、手牌数
2. **次要信息缩小但可访问**: 装备详情、技能描述
3. **隐藏信息有明确标记**: 手牌数(不可见内容)、未公开身份
4. **状态变化有视觉反馈**: 边框颜色、动效、闪烁

### 14.2 布局规律

1. **自己在底部, 对手在上方/两侧** — 符合自然视线
2. **操作区固定在底部** — 操作便利性
3. **信息从上到下递减** — 顶部=远处玩家(少信息), 底部=自己(全信息)
4. **中央区域留给动画/弃牌堆** — 视觉焦点

### 14.3 颜色规律

1. **深色背景突出前景元素** — 暗色牌桌, 亮色卡牌
2. **红/绿表示正负反馈** — 红=伤害/危险, 绿=回复/安全
3. **金色表示重要/当前** — 当前回合/选中状态
4. **势力色贯穿一致** — 边框/标签/背景统一

### 14.4 交互规律

1. **可操作元素有hover反馈** — 边框高亮/放大
2. **不可操作元素降低视觉权重** — 透明度/灰色
3. **选择目标有明确标记** — 红色外描边
4. **倒计时有视觉紧迫感** — 橙色+进度条+数字
