# 引擎缺失能力

本文档记录当前游戏引擎尚不支持的能力,以及每个能力对应的被跳过技能。

> 更新时间:2026-06-14
> 核心结论:**引擎执行模型能表达全部三国杀技能**,没有任何需要打破"同时只一个 pending"不变量的情况。下述缺口全部是数据层(atom/字段)或技能层(execute 逻辑)的加法。

## 执行模型结论

经过对无懈可击、濒死求桃、于吉蛊惑(扣牌+揭示)等"最难"技能的推演验证:

- **多人响应**(求桃/无懈/质疑)都是**依次串行**,不是并发——单槽 pending + `for` 循环 + 状态观察足够表达
- **抢占式响应**(无懈可击)是数据层约定——`请求回应` 声明合法回应者集合(不限单一 target),仍保持同时只一个 pending
- **隐藏信息**(扣牌/展示)用 `toViewEvents` 的 `ownerViews`/`othersView` 分级——不返回给不该看的客户端即可
- **HookResult**(pass/modify/cancel)覆盖了事件取消/变更(仁王盾、藤甲、寒冰剑)的全部需求

因此下述各项**均为数据层缺口**(补 atom、补字段、写技能 execute),不涉及执行模型改动。

---

## 1. 濒死/求桃流程(核心规则)— 数据层缺口

**影响**:几乎每局(桃、酒救援、郭嘉遗计死亡时机、贾诩完杀、周泰不屈)

**现状**:`造成伤害.apply` 扣到 0 直接 `alive=false`,无濒死窗口。

**需要**:
- 新 atom `进入濒死`(target 扣到 0 时触发)
- 濒死是逐个询问的 pending 循环:`for (player of 轮转) await applyAtom(请求回应-桃, target: player)`,有人救则脱离
- `造成伤害` 的 after 钩子(或改为 before 钩子 cancel)扣到 0 时进入濒死循环,无人救才 `击杀`

## 2. 玩家身份/势力 — 数据层缺口

**影响**:主公技(激将、黄天、暴虐)、势力筛选

**现状**:`PlayerState` 无 `identity`/`faction` 字段。`抽身份` atom 存在但身份未存。

**需要**:`PlayerState.identity: '主公'|'忠臣'|'反贼'|'内奸'`、`PlayerState.faction: '魏'|'蜀'|'吴'|'群'`。技能 validate/hook 据此过滤。

## 3. 无懈可击 / 锦囊生效前拦截 — 数据层缺口(抢占式 pending)

**影响**:无懈可击(及其套娃)、所有锦囊的"询问无懈"环节

**现状**:`无懈可击.ts` 注释自承"整体不工作"——锦囊 execute 没有"询问无懈"循环。

**需要**:
- `dispatch` 回应者校验放宽:`请求回应` 加 `respondable?: number[]` 字段(默认 `[target]`),无懈类设为全体活着的玩家;`dispatch:237` 从 `!== target` 改为 `!respondable.includes(ownerId)`
- 锦囊 execute 写无懈轮询循环:`while (true) { applyAtom(请求回应-无懈); if 无人 break; count++; }`,奇偶判定生效
- 多目标锦囊:无懈的 prompt/params 带目标信息,respond 时选(数据层)

## 4. 阶段跳过/替换 — 机制已就绪(HookResult cancel)

**影响**:夏侯渊神速、张郃巧变、颜良文丑双雄、大乔流离

**现状**:阶段开始/结束 atom 有,回合管理硬编码阶段序列。

**需要**:`阶段开始` 的 before hook 返回 `cancel` → 跳过该阶段。`回合管理.ts` 从"硬编码 apply 序列"改为"读 state.phase 驱动,尊重 cancel"。HookResult 机制已就绪。

## 5. 多牌/连续响应 — 数据层缺口(循环 + 单 target pending)

**影响**:吕布无双(需2闪)、貂蝉离间、决斗(轮流出杀)、南蛮/万箭多目标依次响应

**现状**:pending 单槽单 target。决斗已有 `while` 循环雏形(有 bug)。

**需要**:技能 execute 用循环 + 状态观察(同决斗模式,修复 `__决斗回应` 不清理的 bug)。吕布无双 = 连续两次 `询问闪`。

## 6. 觉醒技 — 基础设施已就绪

**影响**:邓艾凿险、孙策魂姿、姜维志继

**现状**:`设上限`、`添加技能`/`移除技能` atom 已实现。

**需要**:`造成伤害` after hook 检查条件(体力降到某值)+ `加标记`(觉醒已触发)。框架就绪,纯技能层。

## 7. 伤害转移 — 机制已就绪(HookResult modify)

**影响**:小乔天香

**现状**:无。

**需要**:`造成伤害` before hook 返回 `{ kind: 'modify', atom: {...atom, target: 新目标} }`。机制已就绪。

## 8. 判定拦截/替换 — 机制已就绪(HookResult modify)

**影响**:张角鬼道(黑牌替换判定牌)、司马鬼才

**现状**:`判定` atom 有。

**需要**:`判定` before hook 用 `modify` 换判定牌。机制已就绪。

## 9. 延时锦囊转化 — 机制已就绪(影子卡牌 + preceding)

**影响**:徐晃断粮(黑基本牌→兵粮寸断)

**现状**:延时锦囊 atom 有,武圣的影子卡牌 + preceding 组合 action 模式已实现。

**需要**:断粮 = preceding(转化黑基本牌为延时锦囊影子)+ 主 action(放置)。同武圣模式。

## 10. 手牌交换 — 数据层缺口(新 atom)

**影响**:鲁肃缔盟

**需要**:新 atom `交换手牌`(两名玩家手牌整体交换)。

## 11. 弃牌阶段拦截 — 数据层缺口(after hook + 取回)

**影响**:张昭张纮固政

**需要**:`弃置` after hook 监听其他角色弃牌 + 新 atom"将弃牌堆顶的牌交给某角色"。

## 12. 多玩家质疑 — 数据层缺口(逐个询问循环)

**影响**:于吉蛊惑

**需要**:
- `扣牌` atom + `展示` atom + `toViewEvents` 分级(隐藏信息)
- 逐个询问质疑:`for (player) await applyAtom(请求回应-质疑, target: player)`
- 揭示后分叉结算(真假分支)

## 13. 创牌机制 — 数据层缺口(新牌区 + 濒死流程)

**影响**:周泰不屈

**需要**:新牌区类型(创牌区)+ 濒死流程的 before hook cancel(替换标准濒死结算)+ 创牌点数去重检查。依赖缺口 1(濒死)。

## 14. 死亡移除技能 — 机制已就绪

**影响**:蔡文姬断肠

**现状**:`移除技能` atom 有。

**需要**:`击杀` after hook + state 记录伤害来源链。依赖缺口 1(濒死确认死亡)。

## 15. 化身系统 — 数据层缺口(牌池数据结构)

**影响**:左慈化身、左慈新生

**现状**:`添加技能`/`移除技能` 有。

**需要**:化身牌池数据结构(未登场武将池 + 化身牌)+ 回合开始时切换化身技能。

## 16. AOE 链式交互 — 数据层缺口(逐个询问循环)

**影响**:贾诩乱武

**需要**:逐个询问 + 限定技标记(整局一次)。同无懈/濒死的循环模式。

## 17. 拼点 — 基础设施已实现

**影响**:荀彧驱虎、太史慈天义、孙策制霸

**现状**:`拼点` atom 有。技能未接入。

---

## 优先级建议

| 优先级 | 能力 | 性质 | 解锁技能数 |
|--------|------|------|-----------|
| P0 | 濒死/求桃 | 数据层(新 atom + 循环) | 全局核心 |
| P0 | 身份/势力 | 数据层(加字段) | 3+ |
| P0 | 无懈可击(广播 pending) | 数据层(dispatch 放宽 + 锦囊循环) | 所有锦囊 |
| P1 | 阶段跳过 | 已就绪(HookResult,适配回合管理) | 3 |
| P1 | 多牌/连续响应 | 数据层(技能循环,修 bug) | 4 |
| P2 | 觉醒/伤害转移/判定拦截/延时转化/死亡移除 | 已就绪或数据层 | 各 1-3 |
| P3 | 其余(手牌交换/创牌/化身/AOE/拼点接入) | 数据层 | 各 1-2 |

**全部为数据层加法或机制已就绪,无执行模型改动。**
