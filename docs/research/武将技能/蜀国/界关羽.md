## 技能详解

### 界武圣（转化技）

> **机制简述**：将红色牌当杀使用，方片杀无距离限制

**官方效果**：你可以将一张红色牌当【杀】使用或打出。你使用的方片【杀】无距离限制。

**实现要点**：
- 组合 action 模型：前端两步 UI（点武圣给红牌加"杀"显示 → 点出杀选目标），提交时一个 ClientMessage：`preceding=[武圣.transform]` + 主 `action=杀.use`。
- 后端 dispatch 先执行 `武圣.transform`（创建影子杀），再 `杀.use` validate 看到"杀"通过。
- 杀技能零感知武圣——它看到的永远是 cardMap 里的一张"杀"。
- 界限突破扩展：红色牌包括装备区（原版仅手牌）。装备区红色牌转化时先卸下到手牌再「当作」。

**实现差异/边界**：
- **方片杀无距离限制未实现**：官方"你使用的方片【杀】无距离限制"——`界武圣.ts` 未实现此距离豁免（与诈降红杀无距离类似的横切机制缺失），转化出的方片杀仍受正常距离校验。

### 义绝（主动技）

> **机制简述**：出牌阶段弃牌令他人展示手牌，黑红触发不同效果

**官方效果**：出牌阶段限一次，你可以弃置一张牌，然后令一名其他角色展示一张手牌。若此牌为：黑色，其本回合非锁定技失效且不能使用或打出手牌，你本回合对其使用的红桃【杀】伤害+1；红色，你获得之，然后你可以令其回复1点体力。

**实现要点**：
- 实际实现为另一套机制（详见下方差异）：出牌阶段弃一张手牌 + 指定一名其他有手牌的角色。
- 目标展示所有手牌（通过 pickProcessingCard prompt 明牌列出给发起者）。
- 发起者选一张 → 目标弃置选中牌（弃置 atom）。
- 若弃置牌花色 ≠ ♥ → 发起者摸一张牌。
- 每回合限一次：`义绝/usedThisTurn`；超时兜底选目标手牌第一张。

**实现差异/边界**：
- **机制完全不同**：实现与官方描述不是同一套义绝规则——
  - 官方：弃牌 → 目标**展示一张手牌** → 按展示牌花色分支（黑色：封非锁定技+禁出牌+红桃杀+1伤；红色：获得该牌+可令其回血）。
  - 实现：弃牌 → 目标**展示所有手牌** → 发起者选一张弃置 → 非 ♥ 则发起者摸一张牌。
- 实现是义绝的**早期/旧版规则**（展示全部+选拆+♥检查摸牌），并非 OL 现行界限突破官方版（展示一张+黑红分支+封技/获得/回血）。该实现应视为**未匹配官方描述**。

**与标版差异**：标版关羽仅"武圣"（红色牌当杀）；界版官方新增"方片杀无距离限制"加强武圣 + 全新"义绝"主动技（替代标版无此技）。**注意**：界武圣的方片杀无距离、义绝的官方分支效果均未实现。
