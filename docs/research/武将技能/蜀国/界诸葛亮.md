## 技能详解

### 观星（主动技）

> **机制简述**：准备阶段观看牌堆顶五张置顶底，全置底可再发动

**官方效果**：准备阶段，你可以观看牌堆顶的五张牌（存活角色数小于4时改为三张），然后将之以任意顺序置于牌堆顶或牌堆底。若皆置于牌堆底，结束阶段你可以再次发动本技能。

**实现要点**：

- 主动技；时机：阶段开始(准备) after-hook
- X = min(全场角色数 state.players.length, 5)
- 询问是否发动 → 询问排列，玩家回应 { top:[...], bottom:[...] }
- 构建 newDeck：`[...bottom, ...middle, ...top.reverse()]`，applyAtom(整理牌堆)
- 牌堆方向约定（与 摸牌 atom 一致）：deck[len-1]=牌堆顶

**实现差异/边界**：**两处未实现**：

1. **「存活角色数小于 4 时改为三张」**：官方明确角色数 < 4 时 X=3；实现固定为 `min(角色数, 5)`，未实现 < 4 时降为 3 的逻辑。

2. **「若皆置于牌堆底，结束阶段你可以再次发动本技能」**：实现未追踪「全置底」状态，未提供结束阶段的二次发动入口。

**与标版差异**：标版观星：X = min(角色数, 5)，无「< 4 改 3」规则，无「全置底再发动」规则。界版新增两个机制，**实现均未覆盖**，目前实现等价于标版观星。

### 界空城（锁定技）

> **机制简述**：无手牌时不能成为杀或决斗的目标

**官方效果**：锁定技，若你没有手牌，你不能成为【杀】或【决斗】的目标。

**实现要点**：

- 锁定技；before-hook 挂「成为目标」（结算阶段第一个 atom）
- 当 target===owner 且 owner 无手牌，且当前结算的牌是 杀 或 决斗 时，cancel 该目标
- 判定「杀/决斗」：优先看 atom.cardId 对应卡牌名；无 cardId 时回退看栈顶帧 skillId
- after-hook 挂「被抵消」：检测自己使用的杀被闪抵消 → 摸一张牌（实现模式与界咆哮完全一致）

**实现差异/边界**：**实现合并了无关效果**：官方界空城仅描述「无手牌不能成为杀/决斗目标」；implNotes 在此技能内**额外实现了「杀被闪抵消后摸一张牌」**——此效果不在官方界空城描述中（疑似与界咆哮/旧版机制混淆）。

空城主体（无手牌不能成为杀/决斗目标）实现与官方一致。
