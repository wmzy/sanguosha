## 技能详解

### 八阵（锁定技）

> **机制简述**：无防具时视为装备八卦阵

**官方效果**：锁定技，若你的装备区里没有防具牌，你视为装备着【八卦阵】。

**实现要点**：
- 复用 `八卦阵.ts` 的「询问闪」before-hook 逻辑（询问是否发动 → 判定 → 红色放虚拟闪 + cancel 主询问闪），仅在 owner 无防具时生效。
- 无防具检查：`equipment['防具']` 为空即无防具。
- 与标版卧龙诸葛共享 `八阵.ts` 实现，机制完全一致。

**实现差异/边界**：实现与官方描述一致。

### 火计（转化技）

> **机制简述**：红色牌当火攻，改展示随机手牌

**官方效果**：你可以将一张红色牌当【火攻】使用。你的【火攻】改为令目标展示随机手牌，你弃置与展示牌颜色相同的牌以造成伤害。

**实现要点**：
- 组合 action 模型（镜像武圣）：前端选红牌 → 选目标 → 提交 `preceding=[火计.transform]` + 主 `action=火攻.use`。
- 后端 dispatch 先执行 `火计.transform`（创建影子火攻），再 `火攻.use` validate 看到"火攻"通过。
- 火攻技能零感知火计——它看到的永远是 cardMap 里的"火攻"。
- 与标版卧龙诸葛共享 `火计.ts` 实现。

**实现差异/边界**：
- **第二段未实现**：官方"你的【火攻】改为令目标展示随机手牌，你弃置与展示牌颜色相同的牌以造成伤害"——界版专属的火攻结算改变（展示随机手牌+弃同色造伤）**未实现**，仍走标版火攻的正常结算（目标自选展示手牌）。
- **牌范围差异**：官方界版"红色**牌**"（含装备区），实现 `火计.ts` validate 限定"红色**手牌**"——装备区红色牌不可用于转化。

### 看破（转化技）

> **机制简述**：黑色牌当无懈且不能被响应

**官方效果**：你可以将一张黑色牌当【无懈可击】使用。你的【无懈可击】不能被响应。

**实现要点**：
- 组合 action 模型（与标版看破一致）：无懈可击窗口期间选黑牌 → 提交 `preceding=[看破.transform]` + 主 `action=无懈可击.respond`。
- 后端 dispatch 先执行 `看破.transform`（创建影子无懈可击），再 `无懈可击.respond` validate 通过。
- 与标版卧龙诸葛共享 `看破.ts` 实现。

**实现差异/边界**：
- **第二段未实现**：官方界版"你的【无懈可击】不能被响应"——界版专属的无懈不可被响应机制**未实现**，转化出的无懈可击仍按标版可被另一张无懈抵消。
- **牌范围差异**：官方界版"黑色**牌**"（含装备区），实现 `看破.ts` 限定"黑色**手牌**"——装备区黑色牌不可用于转化。

**与标版差异**：标版火计/看破限红色/黑色**手牌**；界版官方扩展为"红色/黑色**牌**"（含装备区），并新增火攻结算改变与无懈不可被响应两项加强——**当前实现未覆盖这两项界版专属机制**，本质仍是标版机制。
