## 技能详解

### 驱虎（主动技）

> **机制简述**：与体力高者拼点造伤

**官方效果**：出牌阶段限一次，你可以与体力值大于你的一名角色拼点：若你赢，其对其攻击范围内一名由你选择的角色造成1点伤害；若你没赢，其对你造成1点伤害。

**实现要点**：

- 主动技；出牌阶段限一次，与一名角色拼点
- 拼点规则：A=1(最小), 2-10=面值, J=11, Q=12, K=13；点数大者赢，相等算「没赢」
- 拼点双方各出一张手牌，拼点后两张牌进入弃牌堆
- 赢：该角色对其攻击范围内另一名角色（由荀彧选择）造成 1 点伤害；没赢：该角色对荀彧造成 1 点伤害
- 目标需要 respond 选拼点牌——onInit 时为所有玩家注册 respond action，validate 严格检查 pending requestType
- 限一次：`player.vars['驱虎/usedThisTurn']` + 回合用量 atom 同步 view

**实现差异/边界**：实现与官方描述一致。

### 界节命（被动技）

> **机制简述**：受伤或死亡令他人摸牌弃至上限

**官方效果**：当你受到1点伤害后或死亡时，你可以令一名角色摸X张牌，然后将手牌弃至X张（X为其体力上限且至多为5）。

**实现要点**：

- 被动技；after hook 挂「造成伤害」（target=自己 + amount>0）
- 造成伤害（target=自己 + amount>0）→ 询问发动 → 选目标 → 目标摸牌至上限
- X = min(目标.maxHealth, 5)；补牌数 = X - 目标.hand.length（≤0 则不摸但技能仍算发动）
- 若目标原手牌为 0（摸牌前），荀彧（发动者）摸一张牌
- 目标可选任意存活角色，包括自己；按一次伤害一次触发

**实现差异/边界**：**多处与官方描述不符**：

1. **触发时机**：官方「当你受到 1 点伤害后**或死亡时**」；实现**仅支持受伤触发**，**死亡时触发未实现**。

2. **补牌规则**：官方「令一名角色**摸 X 张牌，然后将手牌弃至 X 张**」（先摸后弃）；实现「令目标**将手牌摸至 X 张**」（只摸不弃，且语义不同——摸至 X 而非摸 X）。

3. **额外效果**：「若目标手牌数原为 0，你摸一张牌」**不在官方描述中**（实现新增）。

**与标版差异**：标版节命：当你受到 1 点伤害后，你可以令一名角色将手牌摸至 X 张（X 为其体力上限）。

界版官方：增加「死亡时触发」+「弃至 X 张」；**界版实现反而最接近标版节命**，仅多了「目标手牌为 0 时荀彧摸 1」的新增效果。
