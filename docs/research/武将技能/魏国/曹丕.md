## 技能详解

### 行殇（被动技）

> **机制简述**：角色死亡时获其所有牌

**官方效果**：当其他角色死亡时，你可以获得其所有牌。

**实现要点**：

- before hook 挂在「击杀」（target≠自己且自己存活）。
- 询问是否发动（请求回应 confirm，FAQ：可选不获得）。
- 获得手牌/装备/判定区的所有牌 → 击杀 apply 继续执行（hand/equip 已空，no-op）。
- 卡牌转移统一走「获得」atom（filter 移除+push 到手牌），判定区牌通过「移除延时锦囊」atom 维护 pendingTricks。
- 击杀 atom 自身会把死亡角色的手牌/装备移入弃牌堆但不处理判定区；行殇先把所有牌拿走，使后续 card-loop no-op。

**实现差异/边界**：官方"其他角色死亡"——实现可处理任意角色死亡（含自己，但 trigger 条件已排除）；获得范围手牌+装备+判定区，与官方"所有牌"语义一致。

### 放逐（被动技）

> **机制简述**：受伤后令他人翻面并摸牌

**官方效果**：当你受到伤害后，你可以令一名其他角色翻面，并令其摸X张牌（X为你已损失体力值）。

**实现要点**：

- after hook 挂在「造成伤害」（target=自己）。
- 选目标（不能是自己）→ 该目标摸 X 张（X=maxHealth-health，已损失体力值）→ 翻面。
- 翻面实现（同据守但 tag 名独立）：加标签 "放逐/翻面" 到目标；before hook 挂「阶段开始」：目标准备阶段开始 + 有翻面标签 → 移除标签 + 设 skipAll 标志 + cancel；其他阶段被 skipAll 一律 cancel；before hook 挂「阶段结束」：skipAll → 主动推进回合（清过期标记 + 下一玩家 + 回合结束）。
- X = maxHealth - health：血越少摸牌越多；翻面 hook 注册在曹丕座次但 hook 对所有玩家阶段 atom 触发，callback 内检查目标 player 是否有翻面标签。

**实现差异/边界**：实现与官方描述一致。翻面持续整一回合（下个回合开始时清除并跳过）。

### 颂威（主公技）

> **机制简述**：魏将黑判定生效后令你摸牌

**官方效果**：主公技，当其他魏势力角色的黑色判定牌生效后，其可以令你摸一张牌。

**实现要点**：

- 主公技，after hook 挂在「判定」（player≠曹丕 + faction=魏 + 判定牌黑色）。
- 询问曹丕是否摸牌（请求回应 confirm）→ 摸牌 1 张。
- 判定牌在 frame.cards 末尾（在判定.afterHooks 移入弃牌堆之前）；黑色 = ♠ 或 ♣；faction 从 player.faction 读取。
- 仅主公曹丕可用（isLord 判定 ownerId===0）；非主公时 hook 注册但不触发。

**实现差异/边界**：实现与官方描述一致。
