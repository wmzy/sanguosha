## 技能详解

### 界奸雄（被动技）

> **机制简述**：受伤害后摸一牌并获得造成伤害的牌

**官方效果**：当你受到伤害后，你可以摸一张牌，并获得造成此伤害的牌。

**实现要点**：

- 造成伤害 after-hook（target=自己）。
- 二选一：① 摸一张牌；② 获得造成此伤害的牌。
- 无来源伤害（闪电等）无牌可获得，只能选①。
- 时序关键（选项②）：伤害牌在造成伤害时位于 frame.cards（处理区），父 execute（杀/万箭齐发/南蛮入侵/决斗）收尾会 applyAtom(移动牌, 处理区→弃牌堆) 入弃牌堆。若奸雄在造成伤害 after-hook 直接拿走，父收尾的 移动牌(to:弃牌堆) 仍会无条件 push 该 cardId → 牌同时出现在手牌与弃牌堆（状态损坏）。
- 故采用"延迟拿取"：造成伤害后记录 wantCard=cardId，挂 移动牌 after-hook，在该伤害牌被移入弃牌堆的瞬间再 移动牌(弃牌堆→手牌)。
- 选项① 摸一张牌无此时序问题，直接 摸牌 count=1。

**实现差异/边界**：实现与官方描述一致。延迟拿取是关键时序处理，避免牌同时出现在两个区域。

**与标版差异**：标版奸雄：受伤后获得造成此伤害的牌。界版：增加"摸一张牌"选项，二选一。

### 护驾（主公技）

> **机制简述**：魏势力可代你用闪，回合外用闪可令你摸牌

**官方效果**：主公技，其他魏势力角色可以在你需要时代替你使用或打出【闪】（然后视为由你使用或打出）。每回合限一次，当其他魏势力角色于其回合外使用或打出【闪】时，其可以令你摸一张牌。

**实现要点**：

- 主公技（曹操为主公时生效，前端 activeWhen 检查 isLord）。
- 注册 respond action 在曹操座次：当前 pending 是询问闪 + target=曹操 → 护驾.respond → 逐个询问魏势力角色出闪。
- 流程：① 曹操被询问闪（询问闪 atom）。② 曹操 dispatch 护驾.respond（pause 询问闪 slot）。③ execute：按座次顺序逐个询问其他魏势力角色是否打出闪（请求回应）；第一个出闪的角色：移闪到处理区（视为曹操打出）；全部拒绝：处理区无闪。④ execute 完成 → dispatch 自动 resolve 询问闪 slot → 杀结算检查处理区。
- 嵌套请求回应创建的 pending slot（魏势力角色）与询问闪 slot 不同 target，不冲突。

**实现差异/边界**：官方第二段"每回合限一次，当其他魏势力角色于其回合外使用或打出【闪】时，其可以令你摸一张牌"——实现仅完成第一段（魏势力代曹操出闪），第二段的"回合外用闪令曹操摸牌"机制未实现。

**与标版差异**：标版护驾仅"魏势力代曹操出闪"。界版增加"回合外用闪令曹操摸牌"——引擎未实现第二段。
