## 技能详解

### 谦逊（锁定技）

> **机制简述**：不能成为顺手牵羊和乐不思蜀的目标

**官方效果**：锁定技，你不能成为【顺手牵羊】和【乐不思蜀】的目标。

**实现要点**：
1. before-hook 拦截（与空城同构——引擎「不能成为目标」类锁定技统一走 effect-level cancel）
2. 乐不思蜀：挂「添加延时锦囊」before-hook cancel 即免疫（放置判定区延时锦囊是乐不思蜀结算必经 atom）
3. 顺手牵羊：挂「获得」before-hook cancel 即不被取牌（获得是顺手牵羊结算必经 atom）
4. 顺手牵羊与反馈/突袭等「获得牌」区分：用 `topFrame(state).skillId === '顺手牵羊'` 精确区分（顺手牵羊.execute 在 获得 期间持有结算帧；反馈顶帧是伤害来源帧如'杀'，不会误判）
5. 被 cancel 后锦囊本身仍按各自 execute 进弃牌堆（牌已消耗），陆逊不受影响——与空城「杀牌仍进弃牌堆、诸葛亮不受伤害」一致

**实现差异/边界**：实现与官方描述一致。因顺手牵羊/乐不思蜀不调用通用「成为目标」atom（与杀/决斗不同），目标合法性仅在各自 use.validate 校验，故谦逊分别挂在这两张牌结算流程中各自必经的 atom 上。

### 连营（被动技）

> **机制简述**：失去最后手牌后摸一张牌

**官方效果**：当你失去最后的手牌后，你可以摸一张牌。

**实现要点**：
1. after hook 挂「移动牌」「给予」「装备」「获得」「弃置」，覆盖所有「手牌减少」路径
2. 触发条件：本次 atom 使自己手牌区牌减少，且 apply 后手牌数 == 0
3. 移动牌/给予/装备：atom 字段（from.zone=手牌）即可判定确实移走手牌 → after-hook 直接判定 hand.length===0
4. 获得/弃置：可能作用于装备/判定区（非手牌），before-hook 记录「涉及手牌」标记，after-hook 仅在标记存在时判定（避免手牌本就为0、只丢装备时误触）
5. 「你可以」→ 询问确认后摸一张牌；无回合限制（自己回合内/外失去最后手牌均可触发）
6. 牌堆+弃牌堆皆空时无法摸牌，提前跳过询问

**实现差异/边界**：实现与官方描述一致。摸牌不减少手牌，不会自递归。
