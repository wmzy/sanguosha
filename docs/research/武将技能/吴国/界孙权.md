## 技能详细规则
### 制衡
- **官方效果**：出牌阶段限一次，你可以弃置任意张牌，然后摸等量张牌，若你以此法弃置了所有手牌，你额外摸一张牌。
- **实现要点**：
  - 主动 use action：出牌阶段限一次（`markOncePerTurn`，键 `界制衡`；与标制衡键隔离）。限一次标记在 execute 第一个 await 之前同步设置（防 fire-and-forget dispatch 在 await 期间重入）。
  - 「任意张牌」= 手牌 + 装备均可弃（标版引擎本就支持，界版沿用）。validate 校验所有 cardId 在手牌或装备区中。
  - 「弃置所有手牌」判定：必须在弃置前快照手牌（applyAtom 弃置会清空）。条件 = 弃置前手牌非空 且 手牌中每一张都在本次弃置集合内（装备可一并弃，不影响判定 —— 只要所有手牌都弃了即满足）。
  - 流程：`弃置 N 张` → `摸牌 count = N + (弃置了所有手牌 ? 1 : 0)`。
  - 独立界版技能文件，不修改标制衡。
- **与标版差异**：
  - 标制衡：弃任意张牌 + 摸等量张，无奖励。
  - 界制衡：新增「弃置所有手牌 → 额外摸 1 张」奖励。

### 救援
- **官方效果**：主公技，其他吴势力角色于其回合内回复体力时，若其体力值大于等于你，则其可以改为令你回复1点体力，然后其摸一张牌。
- **实现要点**：
  - 主公技：仅孙权为主公（`ownerId === 0`，主公固定 0 号位）时生效；非主公座次时 hook 注册但不触发。参考激将/若愚/标救援的主公判定。
  - before-hook 拦截原 `回复体力` atom（参考克己/享乐「before-hook 内 applyAtom 后 return cancel」先例）：
    - 触发对象：回复体力的 target 是「其他吴势力角色」（`target !== ownerId && faction === '吴'`）。
    - 「于其回合内」：`currentPlayerIndex === atom.target`（该角色自己回合）。
    - 「若其体力值大于等于你」：target 当前体力（before-hook 即回复前）≥ 孙权当前体力。
    - 「其可以」：可选，询问 target（回复者）是否发动；选择权在该角色，故 respond（requestType `界救援/choose`）注册到所有吴势力角色座次（同护驾跨座次注册模式）。
    - 「改为」语义（OL FAQ）：该角色本次不回复体力，改为孙权回复 1 点体力，然后该角色摸 1 张。实现：before-hook 内 cancel 原回复 → 对孙权 `回复体力 +1` → 该角色 `摸牌 +1`。
  - 嵌套安全：改对孙权的 `回复体力` `target = 孙权 = ownerId`，本 hook 条件 `target !== ownerId` → 不重入。
  - 与濒死求桃无关：濒死求桃时 target = 孙权（被救者），本 hook 要求 `target !== 孙权`，故天然不与濒死求桃冲突，无需检查 求桃/已救 标志。
- **与标版差异**：
  - 标救援：孙权濒死求桃时，其他吴角色出桃救孙权 → 该救援者额外回复 1 点体力（锁定）。
  - 界救援：其他吴角色【在其自己回合内】回复体力时（任意回复体力事件，非濒死求桃），若其体力 ≥ 孙权，其【可选】改为令孙权回 1 血（替代其原本回复），然后其摸 1 张。

## FAQ / 备注
- 救援「改为」语义：发动后该回复者本次**不**回复体力，改为孙权回复 1 点，然后该回复者摸 1 张。
- 救援触发条件中的「体力值 ≥ 孙权」按 before-hook 时刻（即该角色原本回复体力之前）的体力值判定。
- 救援是主公技，孙权非主公时不生效（与标救援一致）。
- 界救援与濒死求桃场景互不干扰（target 不同），无需特殊兼容。
- 制衡的「弃置所有手牌」奖励判定：装备可一并发动制衡一并弃置，只要所有手牌都包含在弃置集合内即触发额外摸 1 张。
