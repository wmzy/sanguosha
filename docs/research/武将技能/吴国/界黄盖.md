## 技能详细规则
### 苦肉
- **官方效果**：出牌阶段限一次，你可以弃置一张牌，然后失去1点体力。
- **实现要点**：
  - 主动 use action：出牌阶段限一次（`markOncePerTurn`，键 `界苦肉`；与标苦肉键隔离）。
  - 代价分两步：先 `弃置` 一张牌（手牌或装备均可，跨手牌/装备区），再 `失去体力` 1 点（非伤害，不触发伤害技）。
  - 文件名 / loader key / character skill name 均为 `界苦肉`（避开标苦肉冲突）；内部 `Skill.name` 为 `苦肉`（OL 官方技能名，玩家可见）。
- **与标版差异**：
  - 标苦肉：失去 1 点体力 → 摸 2 张，无次数限制、无弃牌代价。
  - 界苦肉：增加「弃置一张牌」代价，限一次/回合；摸牌效果（3 张）拆分到锁定技「诈降」。

### 诈降
- **官方效果**：锁定技，当你失去1点体力后，你摸三张牌，若在你的出牌阶段，你本回合使用【杀】的限制次数+1、使用红色【杀】无距离限制且不能被抵消。
- **实现要点**：
  - 锁定技（不询问）。触发钩点：`失去体力` atom 的 after-hook（`target === ownerId && amount > 0`）。**关键：挂在「失去体力」而非「造成伤害」** —— 失去体力不是受到伤害，二者必须严格区分（苦肉走失去体力，普攻/南蛮走造成伤害）。
  - 效果拆分：
    1. **摸 3 张牌**：无条件，任何阶段失去体力都摸。
    2. **杀增益**（仅当失去体力时处于 owner 自己的出牌阶段才激活，turn-scoped，key `诈降/active`）：
       - a) 杀次数 +1：`registerSlashMaxProvider` 返回 1（base 1 + 1 = 2），**非「无限」**（诸葛连弩/咆哮式 ∞ 由对应 provider 返回 Infinity）。
       - b) 红色杀无距离：杀技能自身 validate 读 `turn.vars['诈降/active']` 且卡色为红时放行（distance.ts 不做全局放行 —— 红色专属，非所有杀）。
       - c) 红色杀不能被抵消：杀技能自身 execute 在询问闪前检查 `turn.vars['诈降/active']` 且卡色为红 → 跳过询问闪（目标不可出闪响应）。
  - 摸 3 与濒死的顺序：引擎 `runAfterHooks` 把系统级 hook（濒死）排最后，故诈降摸 3 先于濒死求桃 —— 黄盖可借诈降摸到的桃自救（OL 经典连招）。
  - turn-scoped：`turn.vars` / `turnUsage` 在回合结束 atom 自动清空，效果仅本回合。杀增益 provider 随技能实例生命周期注册/卸载。
- **与标版差异**：标版无此技能。界诈降是新增锁定技，承接从苦肉拆出的摸牌效果并附加红杀增益。

## FAQ / 备注
- 苦肉 + 诈降连招：出牌阶段发动苦肉 → 弃 1 牌 + 失 1 体力 → 诈降摸 3 + 激活红杀增益（次数+1、红色杀无距离不可抵消）。体力归零时诈降摸 3 先执行，若摸到桃可自救，构成黄盖爆发/濒死自救核心循环。
- 诈降的杀次数 +1 是额度叠加（base 1 + 1 = 2），与诸葛连弩/咆哮的「无限杀」语义不同。
- 红色杀增益仅对【红色】杀生效，黑色杀不享受无距离/不可抵消。
- 诈降对任意「失去体力」事件触发（不仅限于自己的苦肉；若存在其他失去体力来源亦触发）。
