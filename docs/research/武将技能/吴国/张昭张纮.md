## 技能详解

### 直谏（主动技）

> **机制简述**：手牌装备置入他人装备区后摸一张牌

**官方效果**：出牌阶段，你可以将手牌中一张装备牌置入一名其他角色的装备区，然后摸一张牌。

**实现要点**：
1. registerAction use，出牌阶段发动
2. validate：自己回合 + 出牌阶段 + 无阻塞 pending + 存活 + 手牌中有装备牌 + 目标为其他存活角色 + 目标对应装备栏位为空（不得替换）
3. execute：移动牌（自己手牌→目标手牌）→ 装备（目标）→ 添加技能（装备自带技能）→ 摸牌1
4. 装备 atom 的 validate 要求 cardId 在目标手牌中，故先把装备牌移动牌到目标手牌再装备
5. 装备到「他人」而非自己：不能复用 装备通用.use（它绑定 ownerId），手动复刻装备+添加技能序列（目标=target）
6. 无次数限制

**实现差异/边界**：实现与官方描述基本一致。补充限制：目标对应装备栏位须为空（不得替换原装备）——此为实现对「置入装备区」的合理收窄，官方描述未明示替换规则。

### 固政（被动技）

> **机制简述**：他人弃牌阶段结束可还一牌并获其余

**官方效果**：其他角色弃牌阶段结束时，你可以令其获得一张此阶段因弃置进入弃牌堆的牌，然后你可以获得剩余牌。

**实现要点**：
1. 弃置 afterHook：当 `state.phase==='弃牌'` 时，记录该玩家本阶段弃置的牌到 localVars
2. 阶段结束（弃牌）afterHook：对「其他角色」（非自己）若有弃牌记录，询问是否发动固政
3. 确认且弃牌 >1 张：请求回应（固政/选牌）选一张返回该角色
4. 移动牌：所选牌 弃牌堆→该角色手牌；其余牌 弃牌堆→自己手牌
5. respond action 处理确认/选牌两步回应（分支按当前 pending requestType）
6. 仅弃牌阶段弃置的牌计入；非弃牌阶段弃置（制衡/寒冰剑等）被 `state.phase!=='弃牌'` 过滤

**实现差异/边界**：实现与官方描述一致。触发时机依赖「阶段结束(弃牌)」：弃置在「请求回应」pending resolve 后、阶段结束(弃牌)触发前完成，故记录已就绪。
