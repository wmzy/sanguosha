## 技能详细规则
### 好施（界好施）
- **官方效果**：摸牌阶段，你可以多摸两张牌，然后若你的手牌数大于5，你将一半的手牌（向下取整）交给手牌最少的一名其他角色，然后直到你的下回合开始，当你成为【杀】或普通锦囊牌的目标后，其可以交给你一张手牌。
- **实现要点**：
  - 摸牌逻辑（镜像标好施）：摸牌阶段 before-hook +2 张；摸牌后 after-hook 检测手牌数 >5 时分半（向下取整）给手牌最少的一名其他角色。
  - 跨回合被动追踪：给牌后记 `localVars[界好施/受益者]` + `localVars[界好施/被动激活]`；鲁肃下回合开始 after-hook 清除。
  - 被动触发条件：
    - 杀：成为目标 after-hook（atom.target===owner 且卡是杀）。
    - 普通锦囊：请求回应 before-hook（cancelTarget===owner 且卡非延时锦囊）——全体/单目标锦囊在结算前均经「询问无懈可击」（cancelTarget=目标座次）。
  - 触发后询问受益者是否交给鲁肃一张手牌（respond 注册到每个座次：鲁肃处理 confirm/target/give；其他座次处理 passiveGive）。
- **与标版差异**：界版多一个跨回合持续被动——给牌后到鲁肃下回合开始前，鲁肃成为杀或普通锦囊牌目标后，收到好施牌的角色可交给鲁肃一张手牌。摸牌阶段本体（+2、>5 分半给最少者）与标版一致。

### 缔盟（界缔盟）
- **官方效果**：出牌阶段限一次，你可以令两名其他角色交换手牌（两者手牌数之差不大于你的牌数量），若如此做，出牌阶段结束时，你弃置X张牌（X为这两名角色手牌数之差）。
- **实现要点**：
  - use action：选两名角色 → 校验两者手牌数差 ≤ 鲁肃总牌数（手牌+装备，`totalCardCount`）→ 立即交换手牌 → 记录 X。
  - 延迟弃牌：出牌阶段结束 after-hook，鲁肃弃 X 张牌（X=交换时记录的手牌数差），由 `localVars[界缔盟/待弃数]` + `界缔盟/待弃归属` 跨阶段传递。
  - "每回合限一次"：通过 `activeUnlessUsedThisTurn('界缔盟')` 控制 activeWhen。
- **与标版差异**（三处实质差异）：
  1. 前置条件：两者手牌数差 ≤ 鲁肃的牌数（手牌+装备）。
  2. 结算顺序：先交换手牌，后弃牌（标版是先弃后换）。
  3. 弃牌时机：出牌阶段结束时延迟弃（标版是交换前立即弃）。

## FAQ / 备注
- 命名约定：文件名/loader key/character skills 数组中为 `界好施`/`界缔盟`（避免与标版冲突），内部 `Skill.name`（OL 官方技能名）仍为 `好施`/`缔盟`。
- 好施被动仅鲁肃成为杀或普通锦囊牌目标后触发，延时锦囊（闪电/兵粮寸断/乐不思蜀）不触发被动。
- 缔盟前置条件中的"你的牌数量"包括鲁肃的装备区牌（手牌+装备总数）。
- 缔盟延迟弃牌发生在出牌阶段结束（弃牌阶段之前），若鲁肃无足够手牌则按规则弃光。
