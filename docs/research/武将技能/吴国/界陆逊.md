## 技能详细规则
### 谦逊
- **官方效果**：当一张延时锦囊牌或其他角色使用的普通锦囊牌对你生效时，若你为唯一目标，你可以将所有手牌移出游戏直到回合结束。
- **实现要点**（触发型，模式 A）：
  - 两条触发路径：
    1. **`添加延时锦囊` after-hook**：延时锦囊（乐不思蜀/兵粮寸断/闪电）放置到陆逊判定区时触发。延时锦囊天然单目标，无需额外唯一目标判定。
    2. **`成为目标` after-hook**：他人使用的普通锦囊以陆逊为唯一目标时触发。判定：顶帧 `from !== 自己`、顶帧处理区有「普通锦囊」牌（`type === '锦囊牌'` 且非延时/响应）、`frame.params.targets` 恰好 1 且为自己（排除杀/多目标锦囊/自己使用的锦囊）。
  - 询问 `confirm`（requestType `界谦逊/trigger`）；确认后用 `移出游戏` atom 把全部手牌暂存到 `player.vars['界谦逊/移出']`。
  - `回合结束` after-hook：检测到移出区非空 → 用 `归还移出牌` atom 归还手牌。
  - 联动：移出全部手牌会使手牌归零 → 触发界连营，`X = 失去的手牌数 = 移出张数`（界陆逊核心 combo）。
- **与标版差异**：
  - 标谦逊：锁定技，免疫顺手牵羊/乐不思蜀。
  - 界谦逊：全新的触发型可选技（「你可以」），把所有手牌移出游戏至回合结束，**不免疫任何锦囊**。机制完全不同，故新建独立文件，标版保留不动。
- **覆盖边界（诚实记录）**：顺手牵羊/过河拆桥/火攻等直接生效型普通锦囊不走 `成为目标` atom，本实现未覆盖。引擎无统一「锦囊生效」钩点，逐牌挂获得/弃置 before-hook 会破坏这些锦囊的选牌/结算流程，风险高于收益，留作后续工作。当前覆盖：全部延时锦囊 + 经 `成为目标` 流程的单目标普通锦囊（如决斗，及未来任何走该流程的单目标普通锦囊）。

### 连营
- **官方效果**：当你失去所有手牌后，你可以令至多X名角色各摸一张牌（X为你失去的手牌数）。
- **实现要点**（被动触发，模式 A after-hook）：
  - 挂载点覆盖所有「手牌减少」路径：`移动牌 / 给予 / 装备 / 移出游戏` after + `获得 / 弃置` before+after。
  - 触发条件：本次 atom 使自己手牌区牌减少，且 apply 后手牌数 === 0（失去所有手牌）。
  - 「你可以」→ 询问 `confirm`（`界连营/trigger`）；确认后用 `choosePlayer(min=1, max=X)` 选目标（`界连营/select`），对每个目标各 `摸牌(count=1)`。
  - **X 计数机制（关键）**：每个 before-hook 在涉及界陆逊手牌时写入 `界连营/handBefore = apply 前的手牌数`；对应 after-hook 读取并删除该键，作为 X 传给触发逻辑。
    - 移出游戏（界谦逊）一次移走 N 张整手牌 → before 记 N、after 手牌归零 → X = N（联动核心）。
    - 移动牌/给予/装备 单张移走 → X = 1（失去最后一张的边界）。
    - 获得/弃置 可能作用于装备/判定区（非手牌），before-hook 仅在涉及手牌时写入。
  - 「角色」含自己（不排除自己，filter 仅校验存活）；「至多 X 名」是硬规则（max 在引擎侧 clamp）；摸牌不减少手牌，不会自递归；牌堆+弃牌堆皆空 → 无法摸牌，跳过询问。
- **与标版差异**：
  - 标连营：失去最后的手牌时自己摸一张。
  - 界连营：失去所有手牌后令至多 X 名角色各摸一张（X = 失去的手牌数），可分配给多名角色。

## FAQ / 备注
- 核心连招：谦逊将整手牌移出 → 手牌归零触发连营，X = 移出张数 → 令至多 X 名角色（含队友与自己）各摸一张。
- X 的定义为「失去的手牌数」（本次使手牌归零的那次失去事件中失去的手牌张数），不是早期版本的体力值，请勿混淆。
- 谦逊对延时锦囊（乐不思蜀/兵粮寸断/闪电）和经「成为目标」流程的单目标普通锦囊（如决斗）生效；顺手牵羊/过河拆桥/火攻等直接生效型普通锦囊当前实现未覆盖（见上「覆盖边界」）。
- 连营可指定自己作为摸牌目标（不排除自己）。
