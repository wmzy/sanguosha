## 技能详解

### 激昂（被动技）

> **机制简述**：决斗或红杀指定/成为目标后摸一张牌

**官方效果**：当你使用【决斗】或红色【杀】指定目标后，或成为【决斗】或红色【杀】的目标后，你可以摸一张牌。

**实现要点**：
1. after hook 挂「成为目标」一处即可覆盖全部 4 种情形（孙策用/被用 × 红杀/决斗）
2. 红色杀判定：`card.color==='红'`（♥/♦），含转化后的红杀（武圣红牌当杀保留原色）
3. 决斗判定：实卡 `card.name==='决斗'`；离间等无实体虚拟决斗用 `topFrame.skillId ∈ {'决斗','离间'}` 兜底
4. 孙策作为使用者时按 cardId 去重（多目标杀只触发一次）；target 情形天然单一目标一次
5. 「可以摸一张牌」可选，询问 confirm；无次数限制（FAQ 明确）

**实现差异/边界**：实现与官方描述一致。多目标杀（方天画戟/天义）作为使用者时只触发一次（按 cardId 去重），符合「使用一张杀」语义。

### 魂姿（觉醒技）

> **机制简述**：体力为1时减上限并获英姿、英魂

**官方效果**：觉醒技，准备阶段，若你的体力值为1，你减少1点体力上限，然后获得“<font href='712'>英姿</font>”和“<font href='108'>英魂</font>”。

**实现要点**：
1. 觉醒技（强制，无询问），after hook 挂「回合开始」
2. 触发条件：体力≤1 且未觉醒（`player.vars['魂姿/awakened']`）
3. 减1点体力上限（设上限 amount=maxHealth-1；设上限会 clamp 体力，当前体力1 ≤ 新上限3，体力保持1）
4. 永久获得「英姿」「英魂」（添加技能 atom → 系统规则 instantiateSkill，以 ownerId=孙策座次实例化）
5. 觉醒标记 `player.vars['魂姿/awakened']`（后缀不含 usedThisTurn，整局一次，不被「回合结束」清理）

**实现差异/边界**：
- **时序差异**：官方触发时机为「准备阶段」，实现挂在「回合开始」（回合开始早于准备阶段）；由于效果均在出牌前触发且无依赖出牌阶段的状态，实际结算等价
- **条件等价说明**：官方「体力为1」，实现用 `health<=1`；存活玩家体力恒 ≥1，故 `<=1` 与 `===1` 对存活玩家等价

### 制霸（主公技）

> **机制简述**：吴将出牌阶段可与你拼点，没赢你可获拼点牌

**官方效果**：主公技，其他吴势力角色出牌阶段限一次，其可以与你拼点：若其没赢，你可以获得两张拼点牌。若你发动过“魂姿”，你可以拒绝此拼点。

**实现要点**：
1. 主公技变体（盟友主动）：在孙策实例上注册，但 use action 注册在每个吴势力盟友座次上，盟友在自己出牌阶段主动拼点（参考黄天同构模式）
2. 每名角色每回合限一次（`player.vars['制霸/usedThisTurn']`，回合结束自动清空）
3. 拼点点数：A=1, 2-10=面值, J=11, Q=12, K=13；严格大于才算赢，相等算没赢
4. 盟友没赢 → 询问孙策是否获得双方拼点牌（获得则两张牌 处理区→孙策手牌；不获得则→弃牌堆）
5. 若孙策已觉醒（读 `player.vars['魂姿/awakened']` 跨技能共享），可拒绝此拼点
6. 不使用「拼点」atom（其 applyView 固定投影进弃牌堆，与「获得拼点牌」视图冲突），改用「移动牌」逐张流转保持视图一致

**实现差异/边界**：实现与官方描述一致。主公判定沿用 `ownerId===0` 约定（与激将/若愚/黄天一致）。
