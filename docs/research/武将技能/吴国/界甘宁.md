# 界甘宁

## 基本信息
| 属性 | 值 |
|------|-----|
| 名称 | 界甘宁 |
| 势力 | 吴 |
| 体力上限 | 4 |
| 性别 | 男 |

## 技能列表
| 技能名称 | 类型 | 描述（官方逐字） |
|----------|------|------------------|
| 奇袭 | 一般技（转化技） | 你可以将一张黑色牌当【过河拆桥】使用。 |
| 奋威 | 限定技 | 限定技，当一张锦囊牌指定多个目标后，你可以令此牌对任意个目标无效。 |

## 技能详细规则
### 奇袭
- **官方效果**：你可以将一张黑色牌当【过河拆桥】使用。
- **实现要点**：
  - 组合 action 模型（镜像火计/武圣）：前端两步 UI（点奇袭给黑牌加"过河拆桥"显示 → 点过河拆桥选目标），提交时一个 ClientMessage：`preceding=[奇袭.transform]` + 主 action=`过河拆桥.use`。
  - 后端 dispatch 先执行 `奇袭.transform` 创建影子"过河拆桥"（影子 id 为 `${原id}#奇袭`），再由 `过河拆桥.use` 校验，过河拆桥技能零感知奇袭——它看到的永远是 cardMap 里的"过河拆桥"。
  - 黑色牌可来自手牌或装备区（描述明确允许装备区黑色牌）：装备牌需先「卸下」（装备区→手牌，产生 ViewEvent + 清除武器距离 vars），再用「当作」转化，保证 processedView 与 buildView 一致。
  - 原牌归宿：影子过河拆桥入弃牌堆时引擎按 `shadowOf` 还原为原卡，因此无论原牌来自手牌还是装备区，最终都进入弃牌堆。
  - 奇袭出过河拆桥可被无懈可击抵消——由 `过河拆桥.use` 自身的「询问无懈可击」流程保证，与转化来源无关。
- **与标版差异**：奇袭与标版完全相同（标版本就允许黑色装备牌作为来源），直接复用标版奇袭技能。

### 奋威
- **官方效果**：限定技，当一张锦囊牌指定多个目标后，你可以令此牌对任意个目标无效。
- **实现要点**：
  - 触发时机："当一张锦囊牌指定多个目标后" = 多目标锦囊的第一个无懈窗口打开前（before-hook on 「请求回应」）。全体锦囊对第一个目标调用「询问无懈可击」时本 hook 拦截，检测到 frame.skillId 属于多目标锦囊且目标数>1，弹出奋威确认 + 多选面板。同一锦囊只触发一次（cardId 去重标记），后续目标仅检查是否在"无效集合"中。
  - 无效机制：复用全体锦囊（南蛮/万箭/桃园/五谷）的逐目标独立无懈抵消机制——`localVars[无懈/被抵消/${target}]=true` 时该目标被跳过。奋威在 hook 中直接设抵消标记并取消无懈窗口，等价于"令此牌对该目标无效"。
  - 限定技：`player.vars['奋威/used']`，整局一次，不被「回合结束」自动清理。
  - `MULTI_TARGET_SCROLLS` 维护全体锦囊（南蛮入侵/万箭齐发/桃园结义/五谷丰登）的目标计算模式（allOthers / allAlive）。
- **与标版差异**：标甘宁无此技能，奋威为界版新增限定技。

## FAQ / 备注
- 奋威不支持铁索连环：铁索连环采用"整卡一次无懈"（cancelTarget=使用者），不支持逐目标无效，故奋威对其无效。标准全体锦囊场景已覆盖。
- 奇袭原牌进入弃牌堆由引擎 `shadowOf` 还原机制保证，与原牌来源（手牌/装备）无关。
- 奋威多选面板选择"任意个"目标使其无效，可全选也可部分选。
