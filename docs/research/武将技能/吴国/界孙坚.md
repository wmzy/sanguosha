## 技能详解

### 英魂（被动技）

> **机制简述**：准备阶段按失血令角色摸弃牌

**官方效果**：准备阶段，若你已受伤，你可以选择一名其他角色并选择一项：1.令其摸X张牌，然后弃置一张牌；2.令其摸一张牌，然后弃置X张牌。（X为你已损失体力值）

**实现要点**：
- 界孙坚的英魂**直接复用标版英魂**（`src/engine/skills/英魂.ts`，与标版孙坚共享）。
- 机制详见 [孙坚.md](./孙坚.md)：准备阶段 before-hook → 已受伤 → 询问孙坚 → 选目标 → 孙坚自己选方案（摸 X 弃 1 / 摸 1 弃 X）→ 目标摸牌 → 目标自选弃牌。
- `X = 体力上限 - 当前体力值`；目标自选弃牌，弃牌数 clamp 到目标当前手牌数；respond action 注册到每个座次以 `skillId='英魂'` 隔离。

**实现差异/边界**：实现与官方描述一致（与标版英魂完全相同）。

### 武烈（限定技）

> **机制简述**：结束阶段失血令他人获烈标记防伤

**官方效果**：限定技，结束阶段，你可以失去任意点体力，令X名其他角色获得"烈"标记（X为以此法失去的体力值）。当有"烈"的角色受到伤害时，其移除"烈"并防止此伤害。

**实现要点**：
- 时机：「阶段开始（回合结束）」after-hook（孙坚的结束阶段开始）。
- 前置条件：限定技未用 + 孙坚存活 + 体力≥1 + 有其他存活角色 → 询问是否发动。
- 流程：confirm 发动 → 询问失去 N 点体力（`1..当前体力`，通过 hpCount 参数回复）→ 询问选 N 名其他角色（可选目标数 clamp 到存活其他角色数）→ 标记限定技已用 → 失去 N 点体力（可能进入濒死）→ 孙坚存活则给每个目标加 1 个「烈」标记（`mark.id='武烈/烈'`，scope=目标座次）。
- 「烈」效果（锁定）：「造成伤害」before-hook 若 target 有「烈」标记 → 移去 1 枚「烈」（去标记）+ cancel 原伤害。多枚「烈」可叠加（每次受到伤害仅移去 1 枚），用 marks 数组的多条目实现。
- 限定技整局一次：`player.vars['武烈/used']`（永久 vars，不被自动清理）。
- 失去任意点体力：N ∈ [1, 当前体力]；N=当前体力会令孙坚进入濒死（求桃）；若孙坚失血致死且无人救援，后续不发标记。
- 防止伤害 before-hook 作用于任意持有「烈」的角色（不限孙坚的「烈」来源）。

**实现差异/边界**：实现与官方描述一致。N 路由通过 `params.hpCount`（prompt 是 confirm 样式，前端可定制数字选择器）。失血致死的边界（孙坚死亡则不发标记）已正确处理。

**与标版差异**：标版孙坚仅"英魂"单技能；界版官方新增"武烈"限定技（失血发标记防伤），英魂本身机制未变。引擎界孙坚的英魂复用标版实现，武烈为界版独立新增 `武烈.ts`。
