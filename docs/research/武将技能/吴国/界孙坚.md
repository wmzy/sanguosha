## 技能详细规则
### 英魂
- **官方效果**：准备阶段，若你已受伤，你可以选择一名其他角色并选择一项：1.令其摸X张牌，然后弃置一张牌；2.令其摸一张牌，然后弃置X张牌。（X为你已损失体力值）
- **实现要点**：
  - 模式：准备阶段 before-hook 触发（"准备阶段"即"回合开始阶段"）。准备阶段 → 已受伤（HP<上限）→ 询问孙坚是否发动 → 孙坚选目标 → **孙坚自己选方案**（摸X弃1 / 摸1弃X）→ 目标摸牌 → 目标自选弃牌。
  - X = 孙坚体力上限 - 当前体力值（已损失体力值）。
  - 弃牌数 clamp 到目标当前手牌数（手牌不足时弃光）。
  - 目标自己选弃哪些牌（标准规则：弃自己的牌由自己选）。
  - "每回合限一次"：准备阶段每回合仅由「回合管理」派发一次，phase+myTurn 三重门控，无需额外计数标记。
  - 目标可能是任意座次，respond action 注册到每个座次（dispatch 按 skillId+ownerId+actionType 精确路由，以 skillId='英魂' 隔离）。
- **与标版差异**：英魂与标孙坚完全相同（标版 BUG 已于 2026-07 修复——原实现把"选择方案"的决策错给到目标，现已修正为孙坚自己选），直接复用标版「英魂」技能。

### 武烈
- **官方效果**：限定技，结束阶段，你可以失去任意点体力，令X名其他角色获得"烈"标记（X为以此法失去的体力值）。当有"烈"的角色受到伤害时，其移除"烈"并防止此伤害。
- **实现要点**：
  - 时机：阶段开始（回合结束）after-hook——孙坚的结束阶段开始。
  - 流程：
    1. 限定技未用 + 孙坚存活 + 体力≥1 + 有其他存活角色 → 询问孙坚是否发动。
    2. 确认发动 → 询问失去 N 点体力（1..当前体力，通过 hpCount 参数回传）。
    3. 询问选 N 名其他角色（可选目标数 clamp 到存活其他角色数）。
    4. 标记限定技已用（`player.vars['武烈/used']`，整局一次，不被「回合结束」清理）。
    5. 失去 N 点体力（可能进入濒死；若孙坚失血致死且无人救援，后续不发标记）。
    6. 孙坚存活 → 给每个目标加 1 枚「烈」标记（`mark.id='武烈/烈'`，scope=目标座次）。
  - 「烈」效果（锁定）：造成伤害 before-hook——若 target 有「烈」标记 → 移去 1 枚「烈」（去标记）+ cancel 原伤害。多枚「烈」可叠加，每次受到伤害仅移去 1 枚。
  - 失去任意点体力：N ∈ [1, 当前体力]；N=当前体力会令孙坚进入濒死（求桃）。
  - 等量其他角色各获 1 个「烈」：目标数 = min(N, 存活其他角色数)。
  - 防止伤害 before-hook 作用于任意持有「烈」的角色（不限孙坚的「烈」来源）。
  - 模式参考：闭月（结束阶段 after-hook）、天香（造成伤害 before-hook cancel）。
- **与标版差异**：标孙坚无此技能，武烈为界版新增限定技。

## FAQ / 备注
- 英魂已对标版 BUG 修复（标版原把选方案决策错给目标，现已修正为孙坚自己选），界版直接复用修复后的标版「英魂」技能，无独立技能文件。
- 武烈"失去任意点体力"是体力流失（非伤害），N=当前体力会让孙坚进入濒死求桃；若无人救则不发标记。
- 武烈「烈」标记多枚可叠加，每次受伤仅消耗 1 枚，防止该次伤害。
- 武烈限定技整局一次：`player.vars['武烈/used']` 永久 vars，不被「回合结束」自动清理。
