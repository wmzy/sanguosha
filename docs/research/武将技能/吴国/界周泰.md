## 技能详解

### 不屈（锁定技）

> **机制简述**：濒死置创，点数不同则回复至1体力；创牌决定手牌上限

**官方效果**：锁定技，当你处于濒死状态时，你将牌堆顶的一张牌置于你的武将牌上，称为"创"，若此牌点数与其他"创"均不同，你回复至1点体力，否则移去此牌。若你的武将牌上有"创"，你的手牌上限为"创"的数量。

**实现要点**：

- 锁定技；时机：`陷入濒死` after-hook（target===ownerId）
- 流程①（濒死，target===ownerId）：「置创牌」{player=自己}——牌堆顶翻一张：不重复则置于武将牌；重复则移去（进弃牌堆）。共用「置创牌」atom，与标版语义一致
- 读 `localVars['不屈/重复']`：不重复 → 「回复体力」{target=自己, amount=max(0,1-health)} 回复至 1 体力 + 设 `localVars['不屈/存活']=ownerId`（系统规则 runDyingFlow 据此跳过求桃+击杀）；重复 → 不设存活标记，runDyingFlow 继续求桃流程，无人救则击杀
- 手牌上限规则（锁定，随技能实例常驻）：通过 `registerHandLimitProvider` 注册——`player.vars['不屈/创牌']` 非空时手牌上限 = 创牌数量；创牌列表为空时不覆盖（回退默认公式 health+bonus）
- 仅实现官方描述的濒死置创流程；旧实现的「回合结束弃创」无官方依据，已删除

**实现差异/边界**：实现与官方描述一致。

**与标版差异**：

- **回复量不同**：标版不屈成功以 **0 体力存活**（不回复体力）；界版**回复至 1 点体力**（amount=max(0,1-health)）。
- **手牌上限规则**：标版无；界版有创牌时手牌上限 = 创牌数量。
- 二者共用「置创牌」atom（重复时移去此牌，对两版本语义一致）。

### 奋激（一般技（触发技））

> **机制简述**：他人手牌被弃或被获后，可失 1 体力令其摸 2 张

**官方效果**：当一名角色的手牌被弃置或获得后，你可以失去1点体力令其摸两张牌。

**实现要点**：

- 时机：弃置 after-hook + 获得 after-hook，两个 hook 都以 `atom.player` 作为「被弃置/获得的角色的座次」，即奋激目标
- 流程：触发目标 = atom.player（任意角色，含周泰自己）→ 询问周泰是否发动（requestType 含目标座次，以隔离多目标并行触发）→ 确认发动 → applyAtom(失去体力, 周泰, 1) → 若周泰存活则 applyAtom(摸牌, 目标, 2)
- 周泰失去体力可能进入濒死（由系统规则 runDyingFlow 处理；不屈可救）；若周泰因此死亡，后续摸牌不执行（目标无收益）
- 一次性弃置/获得多张牌 = 单个 atom = 单次询问，不重复触发；多个弃置/获得 atom 串行触发，各自独立询问
- 防递归：本技能只触发弃置/获得，自身「失去体力/摸牌」不触发本技能

**实现差异/边界**：实现与官方描述一致。\n\n**裁定**：触发目标可以是任意座次（含周泰本人），不存在势力/敌我限制——与官方「一名角色」表述一致。

**与标版差异**：标版周泰无此技能，**奋激为界版新增**。

## FAQ / 备注

- 命名约定：character skills 数组中为 `界不屈`（避免与标版冲突），文件名/loader key 亦为 `界不屈`。
- 界不屈每次濒死均触发一次置创牌判定；点数重复则该牌进弃牌堆，周泰继续走求桃流程。
- 奋激「失去 1 点体力」是体力流失（非伤害），进入濒死由系统 runDyingFlow 处理；不屈的濒死 after-hook 此时可能再次触发置创判定。
