## 技能详解

### 界红颜（锁定技）

> **机制简述**：黑桃视为红桃；有红桃装备时手牌上限=体力上限

**官方效果**：锁定技，你的黑桃牌和黑桃判定牌视为红桃牌。若你的装备区里有红桃牌，你的手牌上限等于体力上限。

**实现要点**：
1. 判定牌黑桃→红桃：与标版一致（判定 before hook 改花色）
2. 新增：装备区有红桃牌时，手牌上限 = 体力上限（通过 hand-limit 覆盖型 provider 注册，条件生效）
3. 无红桃装备时不返回（走默认公式 = 当前体力），不强行压低上限
4. 与天香的联动：界天香检查 `skills.includes('红颜')||includes('界红颜')`，界小乔拥有界红颜时黑桃手牌/装备也作为合法弃牌

**实现差异/边界**：实现与官方描述一致。

**与标版差异**：标版红颜仅「黑桃视为红桃」；界版追加「装备区有红桃牌时手牌上限=体力上限」——实现正确落地了此追加效果。

### 界天香（被动技）

> **机制简述**：弃红桃防伤并令他人代受（造伤+摸牌 或 失体+获弃牌）

**官方效果**：当你受到伤害时，你可以弃置一张红桃牌防止之并选择一项，令一名其他角色：1.受到伤害来源的1点伤害并摸X张牌（X为其已损失体力值且至多为5）；2.失去1点体力并获得你弃置的牌。

**实现要点**：
1. 时机：造成伤害 before hook（伤害结算前），target=自己 + amount>0 时触发
2. 询问弃一张红桃牌（手牌或装备区；红颜/界红颜下黑桃也视为红桃）+ 选一名其他角色
3. 弃置该红桃牌
4. 询问选择一项：① 该角色受来源1点伤害+摸X ② 该角色失去1点体力+获弃牌
5. 执行所选选项，然后 cancel 原伤害 atom（界小乔不受伤——「防止之」）
6. 选项①：造成伤害{target=所选角色, source=原伤害来源, amount=1}；然后摸X张牌（X = maxHealth - health，至多5；伤害结算后计算）
7. 选项②：失去体力{target=所选角色, amount=1}（非伤害，不触发反馈/奸雄）；然后 移动牌(弃牌堆→该角色手牌)获得本次弃置的红桃牌

**实现差异/边界**：
- 「防止之」= cancel 原伤害，界小乔不受伤（与标版「转移伤害」语义完全不同）——实现正确
- 目标不能是界小乔自己（官方：「令一名其他角色」）
- 选项①的「伤害来源」= 原伤害的来源（保留奖励归属）
- 红颜联动：拥有「红颜」或「界红颜」时，黑桃手牌/装备也作为合法弃牌

**与标版差异**：标版天香为「受到伤害时弃红桃牌将伤害转移给他人，该角色摸X（已损失体力）」；界版官方为「弃红桃防止伤害并令他人二选一（造伤+摸牌 或 失体+获弃牌）」——机制差异显著，实现按界版二选一落地。

### 飘零（主动技）

> **机制简述**：结束判定红桃可置顶或交给角色

**官方效果**：结束阶段，你可以判定，若结果为红桃，你将判定牌置于牌堆顶或交给一名角色，若该角色是你，你弃置一张牌。

**实现要点**：
1. 时机：阶段开始(回合结束=结束阶段) after-hook
2. 询问是否判定（confirm）
3. 判定{player=自己, judgeType='飘零'}（红颜/界红颜会把黑桃判定牌改为红桃）
4. 判定完成后，判定牌已在弃牌堆顶（判定 atom 的 def.afterHooks 已运行）
5. 读弃牌堆顶判定牌：非红桃 → 无效果；红桃 → 继续
6. 红桃：询问是否交给一名角色（confirm）
   - 确认 → 选目标（choosePlayer，含自己）→ 移动牌(弃牌堆→目标手牌)；若目标==自己 → 弃一张手牌
   - 不确认 → 移动牌(弃牌堆→牌堆顶)

**实现差异/边界**：实现与官方描述一致。
