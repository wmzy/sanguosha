## 技能详细规则

### 天义

- **官方效果**：出牌阶段限一次，你可以拼点：若你赢，你本回合使用【杀】的次数+1、无距离限制且可以多指定一个目标；若你没赢，你本回合不能使用【杀】。
- **实现要点**（`src/engine/skills/天义.ts`，标/界太史慈共用）：
  - 出牌阶段限一次（`player.vars['天义/usedThisTurn']`，`/usedThisTurn` 后缀由「回合结束」atom 自动清空），与一名角色拼点；目标需 respond 选拼点牌。
  - 拼点规则（与驱虎/烈刃一致）：A=1（最小），2-10=面值，J=11，Q=12，K=13；点数大者赢，相等算"没赢"。拼点后两张牌进入弃牌堆。
  - **赢的三项效果**（均挂在 turn.vars，回合结束自动清空）：
    1. 攻击范围无限：`turn.vars['天义/win']=发起者座次`，`distance.ts` 的 `inAttackRange` 据此放行。
    2. 可额外使用一张杀：注册 `slashMaxProvider`，win 时贡献 +1（slash-quota 提供者模式）。
    3. 使用杀时可额外指定一个目标：杀.validate 不限目标数（参考方天画戟），配合"攻击范围无限"——一张杀可对任意两名存活角色生效。
  - **输的效果**：`turn.vars['天义/lost']=发起者座次`，注册 `slashBlocker` 返回 true → `canSlash=false` → 杀.use 的 validate 拒绝。无论是否装连弩都禁杀（规则：本回合不能用杀）。
- **与标版差异**：纸面标天义仅"赢则本回合可额外使用一张杀"；界天义额外增加"无距离限制"与"使用杀时可多指定一个目标"。引擎 `天义.ts` 已实现界版全部三项效果，故标/界太史慈复用同一文件，不新建界天义。

### 酣战

- **官方效果**：当你拼点前，你可以令对方用随机手牌拼点。当你拼点后，你可以获得拼点牌中点数最大的【杀】。
- **实现要点**（`src/engine/skills/酣战.ts`）：
  - 两个独立可选效果，均作用于太史慈（酣战 owner）发起的拼点。太史慈的唯一拼点来源是【天义】，故本技挂钩天义的拼点流程。
  - **拼点前·随机手牌拼点**：before-hook 挂「请求回应」。天义询问目标出拼点牌时（`requestType='天义/拼点'` 且为太史慈回合）→ 询问太史慈"是否令对方用随机手牌拼点"。确认则从目标手牌用 `createRng`（基于 `state.rngSeed`，可复现）随机抽一张，写入天义读取的 `localVars['天义/targetCard']`（天义 execute 随后据此把该牌移入处理区），并 cancel 该「请求回应」（目标不再被询问）。酣战与天义同属太史慈，localVars 键共享。
  - **拼点后·获最大点杀**：after-hook 挂「拼点」。拼点 atom 结算后（两张拼点牌已入弃牌堆），若其中含【杀】→ 询问太史慈是否获得"两张拼点牌中点数最大的杀"。确认则取点数最大的那张杀，从弃牌堆移入太史慈手牌。
  - 点数：A=1，2-10=面值，J=11，Q=12，K=13。
  - 拼点发起者判定：`st.currentPlayerIndex===ownerId` 且 `requestType='天义/拼点'`。
  - 两个效果相互独立，各自询问，均可放弃。
- **与标版差异**：酣战为界太史慈新增一般技（标太史慈仅有天义，无酣战）。

## FAQ / 备注

- **天义复用说明**：标/界太史慈共用 `天义.ts`，该文件已包含界版全部效果（次数+1、无距离、多目标），与 hero/463 逐字一致，故 character 文件 `skills` 数组引用标版 `天义` 路径，不新建界天义。
- **酣战"拼点牌中点数最大的杀"**：仅取两张拼点牌中的杀；若两张拼点牌中无杀则不触发获得；若有多张杀取点数最大者。
- **酣战随机手牌拼点**：确认后目标不再被询问选拼点牌，改由系统从其手牌随机抽取一张作为拼点牌；天义据此正常比点结算。
- **天义输的效果**：本回合完全禁杀（slashBlocker），连弩等增加杀次数的手段也无法绕过。
