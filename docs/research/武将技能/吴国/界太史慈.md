## 技能详解

### 天义（主动技）

> **机制简述**：拼点赢则杀次数+1、无距离、多目标；没赢则不能用杀

**官方效果**：出牌阶段限一次，你可以拼点：若你赢，你本回合使用【杀】的次数+1、无距离限制且可以多指定一个目标；若你没赢，你本回合不能使用【杀】。

**实现要点**：
1. 出牌阶段限一次（`player.vars['天义/usedThisTurn']`，回合结束自动清空）
2. 拼点规则：A=1(最小), 2-10=面值, J=11, Q=12, K=13；严格大者赢，相等算「没赢」；拼点双方各出一张手牌，拼点后两张牌进弃牌堆
3. 赢的三项效果（均挂在 turn.vars，回合结束自动清空）：
   - 攻击范围无限：`turn.vars['天义/win']=发起者座次`；distance.ts 的 inAttackRange 据此放行
   - 可额外使用一张杀：注册 slashMaxProvider，win 时贡献 +1
   - 使用杀时可额外指定一个目标：杀.validate 不限目标数（参考方天画戟），配合「攻击范围无限」一张杀可对任意两名存活角色生效
4. 没赢的效果：`turn.vars['天义/lost']=发起者座次`；注册 slashBlocker 返回 true → canSlash=false → 杀.use validate 拒绝（无论是否装连弩都禁杀）
5. respond action 为所有玩家注册（目标需 respond 选拼点牌，validate 严格校验 pending requestType）

**实现差异/边界**：实现与官方描述一致。

### 酣战（被动技）

> **机制简述**：拼点前可令对方随机手牌拼点；拼点后可获最大点杀

**官方效果**：当你拼点前，你可以令对方用随机手牌拼点。当你拼点后，你可以获得拼点牌中点数最大的【杀】。

**实现要点**：
1. 两个独立可选效果，均作用于太史慈（酣战 owner）发起的拼点（太史慈唯一拼点来源是天义，标/界太史慈复用同一天义.ts）
2. 效果1·拼点前随机手牌：before-hook 挂「请求回应」。天义询问目标出拼点牌时（requestType='天义/拼点', 且为太史慈回合）→ 询问太史慈「是否令对方用随机手牌拼点?」。确认则从目标手牌随机抽一张，写入天义读取的 `localVars['天义/targetCard']`，并 cancel 该「请求回应」（目标不再被询问）。耦合说明：酣战与天义同属太史慈，localVars 键共享
3. 效果2·拼点后获最大点杀：after-hook 挂「拼点」。拼点 atom 结算后（两张拼点牌已入弃牌堆），若其中含【杀】→ 询问太史慈是否获得「两张拼点牌中点数最大的杀」。确认则取点数最大的那张杀，从弃牌堆移入太史慈手牌
4. 点数：A=1, 2-10=面值, J=11, Q=12, K=13
5. 随机手牌用 state.rngSeed 的 createRng，保证可复现
6. 两个效果相互独立，各自询问；均可放弃

**实现差异/边界**：实现与官方描述一致。
