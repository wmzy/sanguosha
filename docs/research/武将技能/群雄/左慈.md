## 技能详解

### 化身（被动技）

> **机制简述**：化身武将牌并换其技能

**官方效果**：游戏开始时，你随机获得两张武将牌作为“化身”牌，然后展示其中一张，获得此“化身”牌上的一个技能。回合开始时和回合结束后，你可以替换展示的“化身”牌及因此获得的技能。

**实现要点**：

- 游戏开始（采用"首次任意玩家回合开始"初始化，引擎无 per-player 开局 hook）：从未登场武将随机抽 2 张 → player.vars["化身/牌池"]；亮出第一张（player.vars["化身/亮出"]）→ 选技能 → 添加技能。
- 获得的技能不能是限定技/觉醒技/主公技。
- 回合开始 / 回合结束 after-hook（ownerId 实例）：询问是否更换 → 移除当前技能 → 亮出另一张 → 选技能 → 添加技能。
- 只有一个可选技能时自动选取（无需询问）。
- player.vars 键：化身/牌池、化身/亮出、化身/当前技能。

**实现差异/边界**：简化：① "游戏开始时"由"首次回合开始"近似（主公第一回合开始即视为游戏开始，所有左慈实例同步初始化）；② 完整武将牌+技能选择面板 UI 用 confirm prompt + respond{skill} 实现，精细 UI 待后续完善。技能选择机制与官方一致。

### 新生（被动技）

> **机制简述**：受伤害后获一张新化身

**官方效果**：当你受到1点伤害后，你可以获得一张新的“化身”牌。

**实现要点**：

- 造成伤害 after-hook（target===ownerId 且 amount>0）。
- 询问是否发动（请求回应 confirm），确认则从未登场武将随机抽 1 张 → 追加到 player.vars["化身/牌池"]。
- 无次数限制（每点伤害触发一次）。
- 与化身技能共用 化身/牌池 vars 通道。

**实现差异/边界**：实现与官方描述一致。
