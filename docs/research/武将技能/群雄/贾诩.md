## 技能详解

### 完杀（锁定技）

> **机制简述**：你回合内他人濒死时非濒死者不能用桃

**官方效果**：锁定技，在你的回合内，不处于濒死状态的其他角色不能使用【桃】。

**实现要点**：
1. before-hook 挂「请求回应」(requestType='桃/求桃')：runDyingFlow 按座次依次向每个存活玩家 dispatch 求桃请求
2. 贾诩回合内，被问询者既非贾诩本人、又非当前濒死者时，cancel 该请求（applyAtom 在 before-hook cancel 后直接 return，不创建 pending slot）
3. 钩子注册在贾诩座次，但 请求回应 atom 的 target 是被问询者（随座次变化），handler 内按 atom.target 与「濒死者」判断
4. 濒死者定位：runDyingFlow 执行期间濒死者 alive=true 且 health<=0（击杀 atom 尚未执行），扫描存活且 health<=0 的玩家即当前濒死者
5. 适用范围：桃/酒.respond/急救.respond 等「用桃救援」均通过 桃/求桃 pending 触发，被 cancel 则该角色不能出桃

**实现差异/边界**：实现与官方描述一致。贾诩本人与濒死角色本人不受限制。

### 乱武（限定技）

> **机制简述**：令他人依次选杀最近者或失1点体力

**官方效果**：限定技，出牌阶段，你可以令所有其他角色依次选择一项：1.对其距离最近的另一名角色使用一张【杀】；2.失去1点体力。

**实现要点**：
1. 限定技（整局一次），出牌阶段主动 use；立即标记 `player.vars['乱武/used']=true` 防重入
2. 从贾诩下家起按座次依次遍历每个其他存活角色 P
3. 计算 P 到所有其他存活角色（含贾诩）的有效距离，取最小者 → 「最近角色」集合
4. 请求回应（'乱武/出杀', target=P）：选一张【杀】+ 一名最近角色为目标
5. P 打出有效【杀】+合法目标 → 以 P 为 source 走完整杀结算，不计入出杀次数（技能效果，不调 incSlashUsed）
6. 否则（无杀/超时/pass）→ 失去体力1
7. respond action 注册到每个座次（被问询者非贾诩，须按座次注册才能被 dispatch 路由），onInit 返回合并卸载函数保证清理

**实现差异/边界**：实现与官方描述一致。「另一名角色」含贾诩——贾诩可作为最近目标被指定。距离并列时由该角色自选（前端 targetFilter 提示，后端 respond validate 权威校验）。

### 帷幕（锁定技）

> **机制简述**：不能成为黑色锦囊牌的目标

**官方效果**：锁定技，你不能成为黑色锦囊牌的目标。

**实现要点**：
1. before-hook 拦截黑色锦囊结算流程中「贾诩成为目标/受影响」的关键 atom（与谦逊/空城同构，effect-level cancel）
2. 黑色 = `card.color === '黑'`（♠/♣）
3. 只按黑色锦囊牌颜色统一拦截，不与具体锦囊名耦合：frameIsBlackTrick 判定顶帧 params.cardId 对应的牌是否为黑色锦囊——非锦囊结算帧（如弃牌阶段、反馈/突袭等）顶帧无有效 cardId → false；红色锦囊 → false；黑色锦囊 → true → 拦截
4. 锦囊卡 id 来源：成为目标/造成伤害 由 atom.cardId 直接给出；获得/弃置/设横置 由顶帧 frame.params.cardId 给出

**实现差异/边界**：实现与官方描述一致。统一按颜色拦截，扩展性良好（新增黑色锦囊自动免疫）。
