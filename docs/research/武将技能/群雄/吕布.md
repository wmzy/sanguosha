## 技能详解

### 无双（锁定技）

> **机制简述**：杀需两张闪抵消，决斗需两张杀

**官方效果**：锁定技，你使用的【杀】需两张【闪】才能抵消；与你【决斗】的角色每次需打出两张【杀】。

**实现要点**：
- 你使用【杀】指定一名角色为目标时，该角色需连续使用两张【闪】才能抵消。
- 与你进行【决斗】的角色每次需连续打出两张【杀】。
- 结算逻辑通过辅助函数 `enforceDualDodge` / `enforceDualKill` 嵌入「杀」「决斗」的 execute。
- **采用嵌入 execute 而非纯 hook 拦截的原因**：等待型 atom（询问闪）被 before-hook cancel（如八卦阵判红放虚拟闪）后 after-hook 不触发，无法用 hook 拦截追加第二轮询问。故在 execute 内 `applyAtom(询问闪/杀)` 返回后，由调用方主动检查 source 是否拥有无双，如是则消耗第一张牌并追加一次询问。
- 八卦阵 FAQ 正确支持：八卦阵判定出的闪只算一张，无双下需判两次。第一轮八卦阵判红 → cancel 询问闪 → 杀.execute 仍继续到 `enforceDualDodge` → 发现处理区有虚拟闪 → 消耗 → 追加第二次询问闪 → 八卦阵再判一次。

**实现差异/边界**：实现与官方描述一致。实现细节上为正确支持八卦阵等 hook-cancel 场景，采用嵌入 execute 而非纯 hook 拦截——这是技术实现选择，不影响技能效果。
