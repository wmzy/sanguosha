# 三国杀 AI 配置安装说明

本项目支持 AI 通过 MCP 加入房间与人对局。使用前需安装 Claude Code skill、slash 命令，并配置 MCP server。

## 一、MCP Server 配置

项目根已有 `.mcp.json`，Claude Code 打开本项目时会自动识别并启动 sanguosha MCP server（`tsx src/ai-mcp/server.ts`）。

**前置条件**：三国杀服务端必须运行（`npm run server`，默认端口 3930）。`.mcp.json` 中的 `SGS_SERVER_URL` 默认指向 `ws://localhost:3930/ws`，如服务端在其他地址，修改此 env。

如需为 AI 指定固定座次或房间，可在 `.mcp.json` 的 `env` 中加：
- `SGS_SEAT`：默认座次（默认 0）
- `SGS_PLAYER_ID`：固定玩家 id
- `SGS_ROOM_ID`：启动时自动加入的房间

## 二、安装 Skill 和 Slash 命令

skill 和命令源文件在本项目的 `docs/` 下，不自动生效，需安装到 Claude Code 能识别的位置。

### 方式 A：用户级安装（推荐，所有项目可用）

```bash
# 安装 skill
mkdir -p ~/.claude/skills
cp -r docs/skills/sgs-play ~/.claude/skills/

# 安装 slash 命令
mkdir -p ~/.claude/commands
cp docs/commands/sgs-start.md ~/.claude/commands/
```

安装后：
- skill `sgs-play` 在所有会话中可用，Claude 会自动在相关场景调用。
- 命令 `/sgs-start` 在所有项目中可用。

### 方式 B：项目级安装（仅本项目可用）

```bash
# 安装 skill
mkdir -p .claude/skills
cp -r docs/skills/sgs-play .claude/skills/

# 安装 slash 命令
mkdir -p .claude/commands
cp docs/commands/sgs-start.md .claude/commands/
```

（`.claude/commands/` 目录目前不存在，上述命令会创建它。）

### 一键安装脚本

```bash
bash -c '
  # 用户级安装
  mkdir -p ~/.claude/skills ~/.claude/commands
  cp -r docs/skills/sgs-play ~/.claude/skills/
  cp docs/commands/sgs-start.md ~/.claude/commands/
  echo "✓ skill sgs-play 和命令 /sgs-start 已安装到 ~/.claude（用户级）"
'
```

## 三、使用

### 人机对局流程

1. **启动服务端**：`npm run server`（端口 3930）。
2. **AI 建房**：在 Claude Code 中输入 `/sgs-start`（不带参数），AI 会建房并给出房间码。
3. **人类加入**：浏览器打开游戏首页，点击「多人游戏」，输入房间码加入，点击「准备」。
4. **AI 开局**：双方准备后，AI（房主）自动开局，进入对局。
5. **对局**：AI 按 `sgs-play` skill 的策略自动出牌；人类在浏览器操作。

### 反向：人类建房，AI 加入

1. 人类浏览器「多人游戏」→ 创建房间 → 拿到房间码。
2. AI 输入 `/sgs-start <房间码>` 加入。
3. 双方准备后房主（人类）开局。

## 四、验证

安装后，在 Claude Code 中输入 `/sgs-start` 应能触发命令；调用 `play` 工具应能连上服务端。若连接失败，检查：
- 服务端是否运行（`curl http://localhost:3930/api/rooms`）。
- `.mcp.json` 的 `SGS_SERVER_URL` 是否正确。
- 端口是否被占用。
