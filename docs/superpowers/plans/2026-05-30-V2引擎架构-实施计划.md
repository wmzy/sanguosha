# V2 引擎架构 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 重写游戏引擎为纯函数 `(state, action) → { state, events }` 架构。注册表驱动，零 case 分发。

**Architecture:** 四个注册表（Atom、SkillPhase、Skill、ClientEvent），所有扩展通过 `register*()` 完成。状态完全可序列化，支持 Redis 存储。

**Design Doc:** `docs/superpowers/specs/2026-05-30-V2引擎架构-设计.md`

**Tech Stack:** TypeScript, Vitest

## 关键注意事项

1. **V2 在 `engine/v2/` 独立目录**，不破坏现有 V1 代码。所有测试可在过渡期共存。
2. **所有 register 调用在模块顶层**，通过 import 副作用触发注册。入口文件需要 import 所有 atoms/、phases/、skills/ 模块。
3. **Atom 的 toEvents 中不能调用 applyAtom**（还未 apply）。toEvents 只读取当前 state 生成事件，不修改状态。
4. **resolve() 递归调用自身**处理嵌套 Expr。注意防止无限递归。
5. **trigger 的注册/移除** 必须返回新 state（immutable），不能原地修改 state.triggers 数组。
