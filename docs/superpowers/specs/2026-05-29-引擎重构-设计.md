# 三国杀引擎重构 设计文档

## 概述

基于完整的三国杀规则调研，设计一个高度可扩展的游戏引擎。核心原则：**数据驱动，不是代码驱动**。所有卡牌、角色、技能都是数据定义，引擎只负责解释执行。

## 设计原则

1. **数据驱动** — 新增卡牌/角色只需要添加数据文件，不需要修改引擎代码
2. **组合效果** — 所有效果由基础原语组合而成，支持嵌套和条件分支
3. **钩子触发** — 技能通过注册钩子响应事件，引擎在关键节点调用钩子
4. **管道校验** — 所有操作经过统一校验管道，支持扩展
5. **中断栈** — 杀→闪、锦囊→无懈可击、濒死→桃等中断场景用栈管理

## 架构


## 1. 效果系统

所有卡牌和技能效果由基础原语组合而成。

### 基础效果原语

```typescript
type EffectPrimitive =
  | { type: 'draw'; count: number }
  | { type: 'damage'; amount: number; damageType: '普通' | '雷电' | '火焰' }
  | { type: 'heal'; amount: number; target?: string }
  | { type: 'discard'; source: 'hand' | 'equipment' | 'judge'; count: number; target?: string }
  | { type: 'gainCard'; from: 'deck' | 'discard' | 'player'; source?: string }
  | { type: 'skipPhase'; phase: TurnPhase }
  | { type: 'judge'; condition: (card: Card) => boolean; onSuccess?: Effect; onFail?: Effect }
  | { type: 'convert'; from: CardFilter; to: string }
  | { type: 'redirect'; from: string; to: string }
  | { type: 'addPendingTrick'; trick: PendingTrick; target: string };
```

### 复合效果

```typescript
type Effect =
  | EffectPrimitive
  | { type: 'sequence'; steps: Effect[] }
  | { type: 'conditional'; condition: Condition; then: Effect; else?: Effect }
  | { type: 'repeat'; count: number; effect: Effect };
```

### 效果执行器

```typescript
function executeEffect(game: GameState, effect: Effect, context: EffectContext): GameState;
```

一个函数处理所有效果类型，通过 switch 分发到各原语的实现。

### 卡牌定义（纯数据）

```typescript
interface CardDef {
  name: string;
  type: CardType;
  subtype: CardSubType;
  targetFilter?: TargetFilter;       // 目标选择规则
  effect: Effect;                     // 卡牌效果
  responseWindow?: ResponseWindowType; // 可被什么响应
  usageLimit?: { perTurn?: number };  // 使用次数限制
  range?: number;                     // 攻击范围（武器）
}
```

## 2. 触发系统

引擎在关键节点调用注册的钩子，技能通过钩子响应事件。

### 事件类型

```typescript
type GameEvent =
  | { type: 'turnStart'; player: string }
  | { type: 'turnEnd'; player: string }
  | { type: 'phaseStart'; phase: TurnPhase; player: string }
  | { type: 'phaseEnd'; phase: TurnPhase; player: string }
  | { type: 'cardPlayed'; player: string; card: Card; target?: string }
  | { type: 'becomeTarget'; player: string; card: Card; source: string }
  | { type: 'damageDealt'; source: string; target: string; amount: number; damageType: string }
  | { type: 'damageReceived'; target: string; source: string; amount: number }
  | { type: 'healthChanged'; player: string; from: number; to: number }
  | { type: 'dying'; player: string }
  | { type: 'death'; player: string }
  | { type: 'judgeReveal'; player: string; card: Card; result: boolean };
```

### 钩子系统

```typescript
type HookHandler = (game: GameState, event: GameEvent) => Effect[];

class HookSystem {
  private hooks = new Map<string, HookHandler[]>();

  on(eventType: string, handler: HookHandler): void;
  off(eventType: string, handler: HookHandler): void;
  emit(game: GameState, event: GameEvent): GameState;
}
```

### 技能注册示例

```typescript
// 曹操奸雄
hooks.on('damageReceived', (game, event) => {
  if (event.target === '曹操' && event.card) {
    return [{ type: 'gainCard', from: 'discard', card: event.card }];
  }
  return [];
});

// 甄姬洛神
hooks.on('phaseStart', (game, event) => {
  if (event.player === '甄姬' && event.phase === '准备') {
    return [{ type: 'judge', condition: (card) => card.suit === '♠' || card.suit === '♣' }];
  }
  return [];
});
```

## 3. 响应窗口系统

处理杀→闪、锦囊→无懈可击、濒死→桃等中断场景。

### 响应窗口定义

```typescript
interface ResponseWindow {
  type: 'kill_response' | 'trick_response' | 'dying' | 'aoe_response' | 'duel_response';
  requester: string;           // 谁在等待响应
  validResponders: string[];   // 谁可以响应
  validResponses: string[];    // 可以打出的牌名
  resolve: (responses: Map<string, Card | null>) => GameState;
}
```

### 响应系统

```typescript
class ResponseSystem {
  private stack: ResponseWindow[] = [];

  // 发起响应窗口
  async requestResponse(window: ResponseWindow): Promise<GameState>;

  // 当前是否有待响应的窗口
  hasPending(): boolean;

  // 获取当前需要响应的玩家
  getPendingResponders(): string[];
}
```

### 响应流程（杀→闪）

```
1. 玩家A出杀 → 发起 kill_response 窗口
2. 窗口等待玩家B响应（可出闪或不出）
3. 如果B出闪 → 杀被抵消
4. 如果B不出 → 结算伤害 → 检查濒死
5. 如果濒死 → 发起 dying 窗口（可出桃）
6. 窗口关闭 → 继续游戏
```

## 4. 校验系统

所有操作经过统一的校验管道。

### 校验管道

```typescript
interface ValidationContext {
  game: GameState;
  player: string;
  action: PlayerAction;
  phase: TurnPhase;
}

interface ValidationResult {
  valid: boolean;
  reason?: string;
}

type Validator = (ctx: ValidationContext) => ValidationResult | null;

class ValidationPipeline {
  private validators: Validator[] = [];
  addValidator(v: Validator): void;
  validate(ctx: ValidationContext): ValidationResult;
}
```

### 内置校验器

1. `phaseValidator` — 检查当前阶段是否允许此操作
2. `distanceValidator` — 检查距离限制（杀、顺手牵羊）
3. `targetValidator` — 检查目标合法性
4. `cardLimitValidator` — 检查卡牌使用限制
5. `equipmentValidator` — 检查装备限制

**可扩展：** 新增校验规则只需要添加新的 Validator 函数。

## 5. 判定系统

```typescript
interface JudgeResult {
```

## 实现阶段

### Phase A: 核心引擎
- 状态机（阶段转换）
- 效果系统（基础原语 + 执行器）
- 校验系统（管道 + 内置校验器）
- 距离系统

### Phase B: 卡牌系统
- 卡牌数据定义（所有标准卡牌）
- 卡牌效果配置
- 装备系统

### Phase C: 技能系统
- 触发系统（钩子注册和调用）
- 技能数据定义（所有标准角色）
- 被动技能框架

### Phase D: 响应系统
- 响应窗口（中断栈）
- 杀→闪响应
- 锦囊→无懈可击响应
- 濒死→桃响应

### Phase E: UI 集成
- GameController 适配新引擎
- 前端交互（目标选择、卡牌选择、响应提示）
- 计时器和自动操作
