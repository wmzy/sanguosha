# 游戏日志与重播 设计文档

## 概述

为三国杀游戏添加日志记录和重播功能。采用 Command Log（事件溯源）方案，通过种子化随机数实现确定性重播。

**核心原则:**
- 服务端日志记录完整状态转换操作，用于重建服务端状态
- 玩家日志与服务端发送给客户端的状态变更操作一致，用于恢复客户端状态
- 两者独立维护但一致

## 命名规则

- 函数/变量名: 英文
- 业务操作（技能、卡牌等）: 中文

## 日志数据结构

```typescript
// shared/log.ts

interface GameLog {
  meta: {
    version: string;
    createdAt: number;
    playerCount: number;
    characters: string[];
    seed: number;
  };
  serverOps: Operation[];              // 服务端完整操作序列
  playerOps: Record<string, Operation[]>;  // 每个玩家看到的操作序列
}

interface Operation {
  seq: number;           // 从 0 开始
  timestamp: number;
  type: string;          // 'shuffle' | 'draw' | 'play' | 'damage' | 'heal' | ...
  data: unknown;         // 操作的具体数据
  description: string;   // 人类可读描述（中文）
}
```

### 操作类型

```typescript
type OperationType =
  | 'shuffle'           // 洗牌
  | 'draw'              // 摸牌
  | 'play'              // 出牌
  | 'discard'           // 弃牌
  | 'damage'            // 造成伤害
  | 'heal'              // 回复体力
  | 'equip'             // 装备
  | 'phaseChange'       // 阶段转换
  | 'turnChange'        // 回合切换
  | 'skillActivate'     // 发动技能
  | 'gameStart'         // 游戏开始
  | 'gameEnd';          // 游戏结束
```

### 日志文件格式

保存为 JSON 文件: `sanguosha_{timestamp}_{gameId}.json`

## 种子化随机数

当前 `shared/deck.ts` 的洗牌使用 `Math.random()`。改为种子化随机数，确保重播确定性。

```typescript
// shared/rng.ts

interface Rng {
  next(): number;        // 返回 [0, 1) 的随机数
  nextInt(max: number): number;  // 返回 [0, max) 的整数
}

function createRng(seed: number): Rng {
  // Mulberry32 PRNG
  let state = seed;
  return {
    next() {
      state |= 0;
      state = (state + 0x6D2B79F5) | 0;
      let t = Math.imul(state ^ (state >>> 15), 1 | state);
      t = (t + Math.imul(t ^ (t >>> 7), 61 | t)) ^ t;
      return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
    },
    nextInt(max) {
      return Math.floor(this.next() * max);
    },
  };
}
```

修改 `shuffle` 函数接收 `Rng` 参数:

```typescript
// shared/deck.ts
export function shuffle(cards: Card[], rng: Rng): Card[] {
  const result = [...cards];
  for (let i = result.length - 1; i > 0; i--) {
    const j = rng.nextInt(i + 1);
    [result[i], result[j]] = [result[j], result[i]];
  }
  return result;
}
```

## Logger

在游戏引擎的每个动作点记录日志。

```typescript
// engine/logger.ts

class GameLogger {
  private log: GameLog;
  private seq = 0;

  constructor(meta: GameLog['meta']) {
    this.log = {
      meta,
      serverOps: [],
      playerOps: {},
    };
  }

  // 记录服务端操作
  logServerOp(type: OperationType, data: unknown, description: string): void {
    const op: Operation = {
      seq: this.seq++,
      timestamp: Date.now(),
      type,
      data,
      description,
    };
    this.log.serverOps.push(op);
  }

  // 记录玩家视角的操作
  logPlayerOp(playerName: string, type: OperationType, data: unknown, description: string): void {
    if (!this.log.playerOps[playerName]) {
      this.log.playerOps[playerName] = [];
    }
    const op: Operation = {
      seq: this.log.playerOps[playerName].length,
      timestamp: Date.now(),
      type,
      data,
      description,
    };
    this.log.playerOps[playerName].push(op);
  }

  // 导出日志
  export(): GameLog {
    return this.log;
  }

  // 导入日志
  static import(data: unknown): GameLog {
    // 验证格式
    return data as GameLog;
  }
}
```

### 日志集成点

需要在以下引擎函数中调用 logger:

| 函数 | 服务端操作 | 玩家操作 |
|------|-----------|---------|
| `createGame` | `gameStart` | `gameStart` (每个玩家) |
| `shuffle` | `shuffle` (完整顺序) | — (玩家不需要知道) |
| `drawCards` | `draw` (摸到的牌) | `draw` (摸到的牌, 仅自己) |
| `useCard` | `play` (使用的牌, 目标) | `play` (使用的牌, 目标) |
| `dealDamage` | `damage` (伤害来源, 数值) | `damage` (伤害来源, 数值) |
| `heal` | `heal` (回复数值) | `heal` (回复数值) |
| `changePhase` | `phaseChange` | `phaseChange` |
| `changeTurn` | `turnChange` | `turnChange` |
| `activateSkill` | `skillActivate` (技能名, 详细) | `skillActivate` (公开信息) |
| `checkVictory` | `gameEnd` (获胜身份) | `gameEnd` (获胜身份) |

## ReplayEngine

从服务端操作日志重建完整游戏状态。

```typescript
// engine/replay.ts

class ReplayEngine {
  private log: GameLog;
  private currentStep = 0;
  private states: GameState[];  // 每一步的状态缓存

  constructor(log: GameLog) {
    this.log = log;
    this.states = this.replayAll();
  }

  // 重放所有操作，重建状态
  private replayAll(): GameState[] {
    const states: GameState[] = [];
    const rng = createRng(this.log.meta.seed);
    let state = createInitialState(this.log.meta, rng);
    states.push(state);

    for (const op of this.log.serverOps) {
      state = applyOperation(state, op, rng);
      states.push(state);
    }
    return states;
  }

  // 导航
  next(): GameState { ... }
  prev(): GameState { ... }
  goTo(step: number): GameState { ... }

  // 获取当前状态
  getCurrentState(): GameState { ... }

  // 获取某位玩家在当前步骤的视角
  getPlayerView(playerName: string): PublicGameState { ... }

  // 获取当前操作描述
  getCurrentOp(): Operation | null { ... }

  // 总步骤数
  getTotalSteps(): number { ... }
}
```

## 文件 I/O

### 保存日志

```typescript
function saveLog(log: GameLog): void {
  const json = JSON.stringify(log, null, 2);
  const blob = new Blob([json], { type: 'application/json' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `sanguosha_${log.meta.createdAt}.json`;
  a.click();
  URL.revokeObjectURL(url);
}
```

### 加载日志

```typescript
function loadLog(file: File): Promise<GameLog> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => {
      try {
        const data = JSON.parse(reader.result as string);
        resolve(GameLogger.import(data));
      } catch (e) {
        reject(e);
      }
    };
    reader.onerror = reject;
    reader.readAsText(file);
  });
}
```

## 操作应用函数

重播引擎需要一个 `applyOperation` 函数，根据操作类型更新游戏状态:

```typescript
function applyOperation(state: GameState, op: Operation, rng: Rng): GameState {
  switch (op.type) {
    case 'shuffle':
      // 用 rng 重新洗牌
      return { ...state, deck: shuffle(state.deck, rng) };
    case 'draw':
      // 从牌堆摸牌
      return applyDraw(state, op.data as DrawData);
    case 'play':
      // 出牌
      return applyPlay(state, op.data as PlayData);
    case 'damage':
      // 造成伤害
      return applyDamage(state, op.data as DamageData);
    case 'heal':
      // 回复体力
      return applyHeal(state, op.data as HealData);
    case 'phaseChange':
      // 阶段转换
      return { ...state, phase: op.data as TurnPhase };
    case 'turnChange':
      // 回合切换
      return applyTurnChange(state, op.data as TurnChangeData);
    case 'gameEnd':
      // 游戏结束
      return { ...state, status: '已结束', winner: op.data as Role };
    default:
      return state;
  }
}
```

每个 `apply*` 函数根据操作数据更新状态，与引擎中对应的函数逻辑一致但不记录日志。
