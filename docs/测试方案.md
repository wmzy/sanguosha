# 三国杀游戏测试方案

## 一、概述

本文档描述三国杀游戏的完整测试方案，覆盖选将、出牌、技能、锦囊、刷新等核心游戏流程。测试通过 Browser 工具创建 Debug 房间 (playerCount: 4) 进行实际验证。

### 1.1 测试目标

- 验证游戏核心流程的正确性
- 发现与标准三国杀规则不符的问题
- 确保游戏状态一致性

### 1.2 测试环境

- **服务器**: 本地开发服务器 (localhost:5173)
- **测试工具**: Browser 自动化 (Puppeteer)
- **游戏模式**: Debug 房间，4 人局

## 二、Debug 房间创建方式

### 2.1 协议层

Debug 房间通过 WebSocket 消息创建：

```typescript
// 客户端发送
{ type: 'create_debug_room', playerCount: number }

// 服务端处理 (src/server/app.ts:408-416)
async function handleCreateDebugRoom(playerId: string, playerCount: number, ws: WSContext) {
  if (playerCount < 2 || playerCount > 8) {
    ws.send(serialize({ type: 'error', message: '玩家人数须在2-8之间' }));
    return;
  }
  const room = createDebugRoom(`调试${playerCount}人`, playerCount);
  const session = new GameSession(room, true);  // debug=true
  session.pendingPlayerCount = playerCount;
  gameSessions.set(room.id, session);
  await handleJoinDebugRoom(playerId, room.id, 0, ws);
}
```

### 2.2 游戏初始化流程

```
create_debug_room → createDebugRoom() 
  → GameSession(room, debug=true) 
  → session.startGame(playerCount)
  → create(config) + bootstrap(config)
  → 抽身份 → 选将 → 洗牌 → 发牌 → 第一回合
```

## 五、实际发现的问题（测试运行结果）

### 6.1 高优先级问题 (P1)

| 编号 | 问题描述 | 影响范围 | 发现方式 |
|------|----------|----------|----------|
| P1-01 | **API 不兼容**: `createTestGame()` 返回的 state.players 是数组，但测试代码使用 `state.players.P1` 对象式访问 | 核心数据层 | 单元测试 |
| P1-02 | **API 不兼容**: `state.phase` 实际是 `'出牌'` 而非 `'准备'`，回合流程与预期不符 | 回合流程 | 单元测试 |
| P1-03 | **API 不兼容**: `state.meta.round` 和 `state.meta.turnNumber` 为 undefined | 游戏状态 | 单元测试 |
| P1-04 | **弃牌阶段**: `state.pending?.type === '弃牌阶段'` 无法正确判断弃牌状态 | 回合流程 | 单元测试 |

### 6.2 中优先级问题 (P2)

| 编号 | 问题描述 | 影响范围 | 发现方式 |
|------|----------|----------|----------|
| P2-01 | **LEGACY 代码**: `safeEngine` / `engine` 函数已删除，调用抛出 Error | 多个测试 | 单元测试 |
| P2-02 | **API 不兼容**: `findCardInHand` / `injectCard` 等 helper 函数使用旧 API | 卡牌测试 | 单元测试 |
| P2-03 | **技能系统**: `registerCharacterTriggers` / `getSkillRegistry` 函数不存在 | 技能测试 | 单元测试 |
| P2-04 | **原子操作**: `造成伤害` atom 使用 `state.players[atom.target]` 索引方式，但 players 是数组 | 伤害系统 | 单元测试 |

### 6.3 低优先级问题 (P3)

| 编号 | 问题描述 | 影响范围 | 发现方式 |
|------|----------|----------|----------|
| P3-01 | **测试导入**: 多个测试文件导入已删除的模块 `@engine/mark`, `@engine/phase-advance` 等 | 测试框架 | 集成测试 |
| P3-02 | **测试解析错误**: `tests/server/event-seq.test.ts` 存在语法错误 (await 在非 async 函数中) | 服务器测试 | 集成测试 |
| P3-03 | **E2E 测试**: Browser 自动化测试工具与页面交互存在兼容性问题 | 界面测试 | 手动验证 |
