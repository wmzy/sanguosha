# 添加技能指南

> 本文档是"生产手册"——批量实现三国杀技能前必读。
> 包含:实现规范、代码模板(3 类技能)、测试方法、AI 提示词。

## 一、引擎技能模型(必读)

### 1.1 技能 = 一个模块文件

每个技能是一个 TypeScript 文件 `src/engine/skills/${技能名}.ts`,导出 `SkillModule`:

```ts
export function createSkill(id: string, ownerId: number): Skill;
export function onInit(skill: Skill, ownerId: number): (() => void) | void;  // 后端
export function onMount(skill: Skill, api: FrontendAPI): (() => void) | void;  // 前端 UI
export default { createSkill, onInit, onMount } satisfies SkillModule;
```

然后在 `src/engine/skills/index.ts` 的 `skillLoaders` 注册:
```ts
'技能名': () => import('./技能名'),
```

### 1.2 按玩家实例化

每个技能**按玩家实例化**——`createSkill('杀', 0)` 和 `createSkill('杀', 1)` 是两个独立实例。`ownerId` 是**座次下标**(number,0-based),不是玩家名。

### 1.3 引擎标识约定

| 概念 | 类型 | 说明 |
|---|---|---|
| 玩家/目标/来源 | `number` | 座次下标,`state.players[0]` 是第一个玩家 |
| 技能/卡牌/标记 id | `string` | 如 `'杀'`、`'c1'`、`'酒/nextKillDamageBonus'` |
| localVars key | `string` | 用 `${技能名}/${变量名}` 命名空间,如 `'遗计/分配结果'` |

### 1.4 状态变更的唯一通道

**所有状态变更必经 `applyAtom`**。技能不能直接 mutate `state`——必须通过 `applyAtom(state, atom)` 走完整管线(before hooks → validate → apply → after hooks)。

```ts
import { applyAtom, pushFrame, popFrame } from '../create-engine';
```

### 1.5 跨 atom 通信

**通过 `state.localVars` 观察**,不通过 `frame.params` 突变:

```ts
// 写入(在某 atom 的 after hook 或 execute 中)
state.localVars['遗计/分配结果'] = distribution;

// 读取(在后续 execute 中)
const result = state.localVars['遗计/分配结果'];
```

## 二、四类技能的实现模式

### 模式 A:锁定技(被动触发)

代表:遗计(受伤摸牌)、反馈(受伤拿牌)。通过 `registerAfterHook` 挂在事件 atom 上。

```ts
// 遗计.ts(简化)
import type { AtomAfterContext, Skill } from '../types';
import { applyAtom } from '../create-engine';
import { registerAfterHook, type SkillModule } from '../skill';

export function createSkill(id: string, ownerId: number): Skill {
  return { id, ownerId, name: '遗计', description: '受到伤害后摸两张牌并分配' };
}

export function onInit(skill: Skill, state: GameState): () => void {
  registerAfterHook(state, skill.id, ownerId, '造成伤害', async (ctx) => {
    if (ctx.atom.target !== ownerId) return;  // 只在自己受伤时触发

    await applyAtom(ctx.state, { type: '摸牌', player: ownerId, count: 2 });
    // 询问分配 → 读 localVars → 移牌
    await applyAtom(ctx.state, {
      type: '请求回应', requestType: '遗计/分配', target: ownerId,
      prompt: { type: 'distribute', title: '分配牌', cardIds: [...], minPerTarget: 1, maxPerTarget: 2 },
      timeout: 30,
    });
    const distribution = ctx.state.localVars['遗计/分配结果'] as Array<{ target: number; cardIds: string[] }> | undefined;
    if (distribution) {
      for (const { target, cardIds } of distribution) {
        for (const cardId of cardIds) {
          await applyAtom(ctx.state, { type: '移动牌', cardId, from: { zone: '手牌', player: ownerId }, to: { zone: '手牌', player: target } });
        }
      }
    }
  });
  return () => {};
}

export default { createSkill, onInit } satisfies SkillModule;
```

### 模式 B:主动技(玩家主动发动)

代表:制衡(弃牌换牌)、仁德(给牌)。通过 `registerAction` 注册主动 action。

```ts
// 制衡.ts(简化)
import type { GameState, Json, Skill } from '../types';
import { applyAtom, popFrame, pushFrame } from '../create-engine';
import { registerAction, type SkillModule } from '../skill';

export function createSkill(id: string, ownerId: number): Skill {
  return { id, ownerId, name: '制衡', description: '弃置X张牌,摸X张牌' };
}

export function onInit(skill: Skill, ownerId: number): () => void {
  registerAction(
    skill.id, ownerId, 'use',
    // validate:检查参数合法性
    (state: GameState, params: Record<string, Json>) => {
      const cardIds = params.cardIds as string[] | undefined;
      if (!cardIds?.length) return '请选择要弃置的牌';
      const self = state.players[ownerId];
      if (!cardIds.every(id => self.hand.includes(id))) return '牌不在手牌中';
      return null;
    },
    // execute:通过 applyAtom 驱动状态变更
    async (state: GameState, params: Record<string, Json>) => {
      const cardIds = params.cardIds as string[];
      pushFrame(state, '制衡', ownerId, { cardIds });
      await applyAtom(state, { type: '弃置', player: ownerId, cardIds });
      await applyAtom(state, { type: '摸牌', player: ownerId, count: cardIds.length });
      popFrame(state);
    },
  );
  return () => {};
}

export function onMount(skill: Skill, api: FrontendAPI): void {
  api.defineAction('use', {
    label: '制衡', style: 'primary',
    prompt: {
      type: 'useCard',
      title: '选择要弃置的牌',
      cardFilter: { min: 1, max: 99 },  // 任意手牌
    },
  });
}

export default { createSkill, onInit, onMount } satisfies SkillModule;
```

### 模式 C:转化技(牌包装 + 组合 action)

代表:武圣(红牌当杀)。通过 `preceding` 组合 action + 影子卡牌。详见 §3.1。

```ts
// 武圣.ts(简化)——transform 作为 preceding action
export function onInit(skill: Skill, ownerId: number): () => void {
  registerAction(
    skill.id, ownerId, 'transform',
    // validate:红牌校验
    (state, params) => { /* 检查 cardId 是红色且在手中 */ },
    // execute:创建影子卡(不 mutate 原卡)
    async (state, params) => {
      const cardId = params.cardId as string;
      const orig = state.cardMap[cardId];
      state.cardMap[`${cardId}#武圣`] = { id: `${cardId}#武圣`, name: '杀', suit: orig.suit, rank: orig.rank, type: '基本牌', shadowOf: cardId };
      state.players[ownerId].hand = state.players[ownerId].hand.map(id => id === cardId ? `${cardId}#武圣` : id);
    },
    // rollback:主 action validate 失败时撤销转化
    (state, params) => {
      const cardId = params.cardId as string;
      delete state.cardMap[`${cardId}#武圣`];
      state.players[ownerId].hand = state.players[ownerId].hand.map(id => id === `${cardId}#武圣` ? cardId : id);
    },
  );
  return () => {};
}
```

前端提交时:preceding=[武圣.transform] + 主 action=杀.use(详见 §3.1)。

### 模式 D:防具/武器拦截(before hook + HookResult)

代表:仁王盾(黑杀无效→cancel)、藤甲(减伤→modify)、寒冰剑(改为弃牌→cancel)。

```ts
// before hook 返回 HookResult 干预当前 atom
registerBeforeHook(state, skill.id, ownerId, '造成伤害', async (ctx): Promise<HookResult | void> => {
  if (ctx.atom.target !== ownerId) return;
  // ... 检查条件 ...
  return { kind: 'cancel' };   // 取消伤害(仁王盾)
  // 或 { kind: 'modify', atom: { ...ctx.atom, amount: newAmount } };  // 减伤(藤甲)
});
```

## 三、测试方法

### 3.1 用 SkillTestHarness

```ts
import { SkillTestHarness } from '../engine-harness';
import '../../src/engine/atoms';
import '../../src/engine/skills';
import { createGameState } from '../../src/engine/types';

const harness = new SkillTestHarness();
await harness.setup(createGameState({
  players: [
    { index: 0, name: 'P1', character: '关羽', health: 4, maxHealth: 4, alive: true, hand: ['c1'], equipment: {}, skills: ['杀', '武圣'], vars: {}, marks: [], pendingTricks: [], judgeZone: [] },
    { index: 1, name: 'P2', character: '曹操', health: 4, maxHealth: 4, alive: true, hand: [], equipment: {}, skills: ['闪'], vars: {}, marks: [], pendingTricks: [], judgeZone: [] },
  ],
  cardMap: { c1: { id: 'c1', name: '过河拆桥', suit: '♥', rank: 'A', type: '锦囊牌' } },
  currentPlayerIndex: 0, phase: '出牌', turn: { round: 1, phase: '出牌', vars: {} },
}));

const P1 = harness.player('P1');  // 或 harness.player(0)
const P2 = harness.player('P2');

// 转化技:武圣红牌当杀
await P1.transformThenUse('武圣', { cardId: 'c1' }, '杀', { cardId: 'c1#武圣', targets: [1] });
await P2.pass();  // P2 不闪

expect(harness.state.players[1].health).toBe(3);
```

### 3.2 常用操作

| 方法 | 用途 |
|---|---|
| `P1.useCardAndTarget('杀', 'c1', [1])` | 出杀指定目标 |
| `P1.useCard('桃', 'c4')` | 出无目标牌 |
| `P2.pass()` | 不回应(等同超时) |
| `P2.confirm(true/false)` | 确认/取消(八卦阵等) |
| `P1.transformThenUse(...)` | 转化技(武圣+杀) |
| `P1.expectPending('询问闪')` | 断言当前 pending |
| `P1.expectAtoms('造成伤害', '击杀')` | 断言事件流(子序列) |
| `P1.findValidCard('use')` | 跑真实 cardFilter 找合法牌 |

## 四、实现 Checklist

每个技能实现完,逐项检查:

- [ ] `createSkill` 返回正确的 `name`/`description`
- [ ] `onInit` 的 `ownerId` 参数是 `number`(座次),函数体内用座次比较
- [ ] 所有状态变更走 `applyAtom`,不直接 mutate state(除了影子卡创建)
- [ ] 跨 atom 通信走 `state.localVars`,不 mutate `frame.params`
- [ ] `validate` 检查参数合法性(cardId 类型、目标存在、手牌持有等)
- [ ] `onMount` 的 `defineAction` prompt 与后端 validate 一致(前端 filter 是 validate 的超集)
- [ ] use action 的 `cardFilter` 正确声明适用牌范围——**前端靠 filter 匹配选中卡,不靠 card.name→skillId 反查**(装备等通用技能的 use action 尤其要声明准界)
- [ ] 如有特殊激活时机(非缺省的「出牌阶段+自己回合+无 pending」),在 `defineAction` 声明 `activeWhen`;不要依赖组件硬编码分支
- [ ] 转化技用 preceding + 影子卡(不 mutate cardMap)
- [ ] 在 `skills/index.ts` 的 `skillLoaders` 注册
- [ ] 写测试(`tests/skill-tests/${技能名}.test.ts`),至少覆盖 happy path

## 六、通用机制设计(§1.6)

### 1.6.1 出杀次数(turn.vars['杀/quota'])

**问题**:诸葛连弩、咆哮(张飞)、方天画戟等都涉及"出杀次数"。每个技能自造标签会导致协议碎片化。

**统一机制**:
- `state.turn.vars['杀/quota']` = 本回合剩余出杀次数(number,默认 1,回合开始时重置)
- 杀.validate:`quota = turn.vars['杀/quota'] ?? 1;if (quota <= 0) return '出杀次数已用尽'`
- 杀.execute 末尾:`turn.vars['杀/quota'] = quota - 1`
- 增加次数的技能(诸葛连弩/咆哮):after hook 监听 `阶段开始(出牌)`,`turn.vars['杀/quota'] = Infinity`
- "出杀后+1次"类技能:after hook 监听 `造成伤害`(source===自己),`turn.vars['杀/quota'] += 1`

### 1.6.2 装备技能加载/卸载

**机制**:`装备通用.execute` 在装装备时 `applyAtom(添加技能)`,卸下时 `applyAtom(移除技能)`。
技能的动态注册/注销由引擎自动完成,技能本身不需要处理加载时机。

**顺序约束**:卸下装备时,`卸下` atom 的 after hook(如白银狮子失去回血)必须在 `移除技能`**之前**触发。
装备通用的 execute 必须保证:先 applyAtom(卸下)→ 再 applyAtom(移除技能)。

### 1.6.3 横切规则(防具穿透等)

**原则**:涉及"所有同类技能都需检查"的规则,用标签统一,不要每个技能手动加 if。

**当前实现**:青釭剑给目标加 `tag:青釭剑/无视防具`,所有防具 before hook 首行检查:
```ts
if (state.players[ownerId].marks.some(m => m.id === 'tag:青釭剑/无视防具')) return;
```
**替代方案**(更优,待评估):在引擎层 before hook 折叠阶段,检测 ignoreArmor 标签自动跳过防具类 hook。
这需要 hook 分类机制(防具 hook 标记为 armor-type),引擎折叠时遇 ignoreArmor 跳过所有 armor-type hook。

### 1.6.4 身份/势力

`PlayerState.identity`('主公'|'忠臣'|'反贼'|'内奸')和 `PlayerState.faction`('魏'|'蜀'|'吴'|'群')。
主公技 validate 检查 `state.players[ownerId].identity === '主公'`,势力相关技能检查 faction。
