# 游戏引擎完整重构计划

## 目标

将三国杀引擎从"God Class + 硬编码"架构重构为"数据驱动 + 效果原语 + 钩子触发"架构。删除旧代码，建立新系统。

## Phase 1: 共享类型增强

### 1.1 Card 增加 id 字段
```typescript
interface Card {
  id: string;  // 唯一标识，格式 "杀♠7" 或 UUID
  name: string;
  // ... 其他字段不变
}
```
- 目的：消除 name+suit+rank 匹配的 bug
- 修改 shared/cards.ts 的 createDeck()，为每张牌生成 id

### 1.2 GameState 携带 Rng
```typescript
interface GameState {
  // ... 现有字段
  rng: Rng;  // 种子化随机数生成器
}
```
- 目的：所有随机操作统一走 Rng，消除 Math.random()
- 修改 state.ts 的 createGame，将 rng 存入 state

### 1.3 完善 Effect 类型
- 移除 EffectPrimitive 中不合理的字段（如 string 类型的 condition/failEffect）
- 增加必要的字段使所有 25 个角色的技能都可以用数据描述
- 保持向后兼容现有 CardDef 和 CharacterConfig 定义

### 1.4 完善事件类型
- 定义结构化的 GameEvent 类型（替代 data: unknown）
- 覆盖所有触发时机：damageReceived、cardPlayed、turnStart、death 等

## Phase 2: 引擎核心系统（engine/）

### 2.1 engine/state.ts — 状态管理
- createGame(): 创建初始状态（含 Rng、牌堆洗牌、角色分配）
- startGame(): 发初始牌
- getPlayer()、getAlivePlayers()、checkVictory()、playerDeath()
- getPublicState(): 生成玩家可见状态
- updatePlayer(): 统一的玩家状态更新辅助函数

### 2.2 engine/turn.ts — 回合管理
- phaseOrder 定义
- nextPhase(): 阶段推进
- drawPhase(): 摸牌（含牌堆耗尽处理，统一走 Rng）
- handleJudgePhase(): 判定阶段处理延时锦囊
- checkDiscard()、executeDiscard(): 弃牌

### 2.3 engine/effect.ts — 效果执行器（核心）
- executeEffect(game, effect, context) -> GameState
- 完整实现所有 EffectPrimitive：
  - draw: 摸牌
  - damage: 造成伤害（触发 onDamageReceived 钩子）
  - heal: 回复体力
  - discard: 弃牌
  - gainCard: 获得牌（从弃牌堆/牌堆/其他玩家/判定牌/伤害牌）
  - skipPhase: 跳过阶段
  - skipDraw: 跳过摸牌
  - judge: 判定（翻牌判定，根据条件执行成功/失败效果）
  - addPendingTrick: 放置延时锦囊
  - convert: 卡牌转化
  - redirect: 转移目标
  - giveCards: 给牌
  - lookAtTopCards: 观星
  - dealDamage: 造成伤害（含条件加伤）
  - sequence: 顺序执行
  - conditional: 条件执行

### 2.4 engine/trigger.ts — 触发系统
- TriggerSystem 类（基于 core/trigger.ts）
- on/off/emit 接口
- emit 执行所有匹配的 handler，收集 effects，调用 executeEffect
- 按触发类型索引 handler（性能优化）

### 2.5 engine/skill.ts — 技能系统
- registerCharacterSkills(): 从 CharacterConfig 注册被动技能到 TriggerSystem
- getAvailableSkills(): 获取当前可用主动技能
- checkCondition(): 完整实现所有 Condition 字段检查

### 2.6 engine/response.ts — 响应窗口系统
- ResponseSystem 类（基于 core/response.ts，同步模型）
- ResponseWindow 定义
- 预构建的工厂函数：
  - createKillResponseWindow(): 杀→闪
  - createAOEResponseWindow(): 万箭→闪 / 南蛮→杀
  - createDuelResponseWindow(): 决斗→杀
  - createDyingResponseWindow(): 濒死→桃
  - createTrickResponseWindow(): 锦囊→无懈可击
- 嵌套支持（无懈可击反制无懈可击）

### 2.7 engine/validation.ts — 校验系统
- ValidationPipeline 类
- 内置校验器：phase、distance、target、cardLimit、equipment
- 可扩展的校验管道

### 2.8 engine/judge.ts — 判定系统
- performJudge(): 翻牌判定
- resolvePendingTricks(): 回合开始处理延时锦囊
- 乐不思蜀/兵粮寸断/闪电 各自的判定逻辑

### 2.9 engine/distance.ts — 距离计算
- getDistance(): 距离计算（含装备修正）
- getAttackRange(): 攻击范围
- getValidTargets(): 合法目标

## Phase 3: 新 GameController

### engine/game.ts — 薄协调器（~200-300行）
```typescript
class GameController {
  private state: GameState;
  private triggers: TriggerSystem;
  private responses: ResponseSystem;
  private logger: GameLogger;

  // 初始化
  static createGame(characters, seed?, logger?): { state, controller }

  // 核心操作（统一入口）
  playCard(player, cardId, target?): ActionResult
  endTurn(player): ActionResult
  discard(player, cardIndices): ActionResult
  respondToWindow(player, cardId?): ActionResult
  activateSkill(player, skillIndex): ActionResult

  // 查询
  getState(): GameState
  getPublicState(observer): PublicGameState
  getValidActions(player): ValidActions
  getCurrentResponseWindow(): ResponseWindow | null
}
```

关键流程：
1. playCard() → validation → lookup CardDef → executeEffect → trigger hooks → check response windows → check dying
2. respondToWindow() → pop from ResponseSystem → resolve → continue flow
3. endTurn() → check discard → nextPhase → handlePendingTricks → drawPhase → advanceToPlayPhase

- Phase 2.1-2.3（state/turn/effect）顺序执行
- Phase 2.4-2.8（trigger/skill/response/validation/judge/distance）可以并行

- Phase 4（测试）依赖 Phase 3

## 关键约束

1. **PRNG 一致性**：所有随机操作（洗牌、随机选牌、判定）统一走 state.rng
2. **不可变状态**：所有函数返回新 GameState，不修改原对象
3. **数据驱动**：新增卡牌/角色只需添加数据，不改引擎代码
4. **同步响应模型**：ResponseSystem 是同步的，不使用 Promise
5. **卡牌 ID 唯一**：所有卡牌通过 id 标识，不用 name+suit+rank 匹配
